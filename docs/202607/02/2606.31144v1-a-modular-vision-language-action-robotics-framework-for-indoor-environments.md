## 摘要
本文介绍了为CMU视觉-语言-动作（VLA）挑战赛设计的集成系统，旨在使自主智能体能够基于自然语言指令执行复杂任务。我们的框架采用模块化架构，协调环境映射、问题处理与导航。系统以两个并行流运行：感知管道利用OwlViT嵌入从实时摄像头数据构建语义体素地图，语言管道通过视觉-语言模型对用户指令进行分类。映射受时间限制：若达到500秒探索上限，系统将基于部分地图继续执行。随后，分类后的查询被置于地图的几何与语义背景中，为VLM生成详细提示，从而产生可执行的输出，展示了弥合人类语言与机器人动作之间差距的可行方案。

## Abstract
This paper presents an integrated system for the CMU Vision-Language-Action (VLA) Challenge, designed to enable an autonomous agent to perform complex tasks based on natural language instructions. Our framework employs a modular architecture that orchestrates environment mapping, question processing, and navigation. The system operates in two parallel streams: a perception pipeline that constructs a semantic voxel map from real-time camera feeds using OwlViT embeddings, and a language pipeline that classifies user commands with a Vision-Language Model. The mapping is time-constrained; the system proceeds with a partial map if a 500-second exploration limit is reached. The classified query is then grounded in the geometric and semantic context of the map to generate a detailed prompt for the VLM. This yields an actionable output, demonstrating a capable solution for bridging the gap between human language and robotic action.

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：采用**双流水线模块化架构**，将环境感知与语言理解解耦：
  - **感知流水线**：实时构建**语义体素地图**，存储物体的3D坐标、尺度以及由OwlViT生成的高维特征嵌入（CLIP风格）。
  - **语言流水线**：由Gemini 2.0 Flash VLM对用户指令进行分类（数值、物体引用、指令跟随），然后通过**接地**过程（计算查询文本嵌入与体素地图嵌入的相似度）定位相关物体，最后生成**上下文丰富的提示**送入VLM进行推理。
- **关键技术细节**：
  - **自主探索**：基于LiDAR构建2D网格地图，规划器生成全覆盖路径（采样遍历点并计算最优游览序列）。探索限时**500秒**，超时则使用部分地图。
  - **语义体素地图迭代构建**：每个物体检测结果（来自OwlViT）的嵌入与坐标逐步插入体素地图，支持部分建图。
  - **提示生成与动作执行**：根据指令类别，VLM输出结构化的动作计划（如Goto(x,y), Avoidbetween），后处理模块解析并驱动导航（路径规划、动态障碍标记）。
- **算法流程**（文字描述）：
  1. 启动系统，创建2D网格地图和空语义体素地图。
  2. 规划器生成全覆盖遍历路径，机器人开始自主探索。
  3. 同时，感知节点通过OwlViT处理实时摄像头画面，将检测到的物体信息（嵌入+坐标）增量更新到体素地图。
  4. 若500秒内完成探索，使用完整地图；否则使用当前部分地图。
  5. 接收用户指令，送Gemini分类。
  6. 提取指令中的物体名词，计算文本嵌入与体素地图的相似度，定位物体并获取坐标。
  7. 将物体信息（名称、坐标、尺度、网格位置）组装成上下文提示，再次送Gemini推理，得到可执行计划。
  8. 后处理模块解析计划，调用导航堆栈执行动作（发布标记、计算路径或动态修改地图）。

### 6. 论文的主要结论与发现

- **主要结论**：结合OwlViT的语义特征提取与Gemini的高级推理能力，模块化的“探索-建图-查询-动作”范式能够有效处理VLA挑战中的多样化指令。时间约束下的部分地图策略保证了响应性。
- **关键发现**：
  - 无记忆的端到端方法（如BLIP-2单帧）不可行，必须构建结构化世界模型。
  - 参数优化（增大路径点间距、放宽到达条件、增大体素大小）可大幅减少探索时间（50%），且不影响物体定位的准确性。
  - 利用VLM做查询分类和接地是可行的，但依赖在线API带来延迟和可靠性问题。
