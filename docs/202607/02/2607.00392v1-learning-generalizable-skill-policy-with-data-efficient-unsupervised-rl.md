## 摘要
无监督强化学习旨在无需外在奖励的情况下预训练可扩展的、以技能为条件的策略，作为下游控制任务的基础。尽管近期取得了进展，我们认为当前离策略的无监督强化学习方法受到两个关键且被忽视的瓶颈限制：（1）非平稳的技能语义和（2）脆弱的泛化能力。为解决这些挑战，我们提出了GenDa（可泛化的数据高效智能体），一个用于鲁棒无监督强化学习的统一框架。首先，我们引入了一种技能重标记机制以缓解非平稳性，并显著提高预训练的数据效率。其次，我们提出了一种互补信息瓶颈，鼓励学到的技能策略关注以自我为中心的特征，并对下游任务的分布偏移具有鲁棒性。通过多种实验，我们证明GenDa显著增强了无监督强化学习的可扩展性，具有优越的泛化能力和数据效率。我们的代码和视频可在https://ihatebroccoli.github.io/official-GenDa获取。

## Abstract
Unsupervised Reinforcement Learning (URL) aims to pre-train scalable, skill-conditioned policies without extrinsic rewards, serving as a foundation for downstream control tasks. Despite recent progress, we argue that current off-policy URL methods are limited by two critical, overlooked bottlenecks: (1) non-stationary skill semantics and (2) brittle generalization. To address these challenges, we propose GenDa (Generalizable Data-efficient Agent), a unified framework for robust unsupervised reinforcement learning. First, we introduce a skill relabeling mechanism to mitigate non-stationarity and significantly improve data efficiency for pre-training. Second, we propose a Complementary Information Bottleneck (CIB), encouraging the learned skill policy to focus on ego-centric features and become robust to distribution shifts for downstream tasks. Through various experiments, we demonstrate that GenDa significantly enhances the scalability of URL with superior generalizability and data efficiency. Our code and videos are available at https://ihatebroccoli.github.io/official-GenDa.

- **核心问题**：当前无监督强化学习（URL）中，离策略方法存在两个关键瓶颈：
  1. **非平稳技能语义**：在离策略训练中，同一技能向量 z 在不同训练阶段产生的轨迹可能不同，导致回放缓冲区中的 `(z, 轨迹)` 对语义漂移，使策略更新不稳定、样本效率低下。

- **整体含义**：为构建可扩展的、可迁移的无监督预训练基础策略，必须同时解决数据效率（语义漂移）和鲁棒泛化（全局上下文解耦）问题。GenDa 框架通过技能重标记和互补信息瓶颈，系统性地缓解了这两个瓶颈，使 URL 能更高效地预训练出可复用于多种下游任务的技能策略。

### 方法论：核心思想、关键技术细节

1. **技能重标记（Skill Relabeling）**  
   - **动机**：缓解语义漂移，即回放缓冲区中固定的 `(τ, z_roll)` 对随表示函数 φ 的演化而失效。
   - **对表示学习的重标记（z-step → ϕ-step）**：
     - 对每条轨迹 τ（端点状态 s₀, s_T），根据当前 φ 计算重标记后的技能向量：  
       `z_relab = unit(ϕ(s_T) − ϕ(s₀))`（应用 stop-gradient）。
     - 在 ϕ-step 中，使用 z_relab 代替原始 z_roll 优化表示目标（式 3），实现语义对齐。
   - **防止坍塌的均匀性正则化**：
     - 由于重标记后的 z_relab 分布不固定，引入对比式均匀性损失 `L_unif`（式 5），鼓励批内 z_relab 向量均匀分布在超球面上，最大化熵 H(z_relab)，从而维持技能多样性。
   - **对策略学习的重标记**：
     - 内在奖励函数 `r(ϕ, z) = (ϕ(s′) − ϕ(s))^T z` 允许利用状态对（s, s′）产生多种目标 z。
     - 对每个转移（s, s′），额外采样步长 c，计算 `z_c = unit(ϕ(s_{t+c}) − ϕ(s_t))`，并与 z_roll、z_relab 混合，增强策略和价值函数可学习的信号多样性。

2. **互补信息瓶颈（Complementary Information Bottleneck, CIB）**  
   - **动机**：使技能策略 π 不依赖全局上下文，而聚焦于局部、自我中心的特征。
   - **核心思想**：训练编码器 `q_ψ(ℓ | s)` 与解码器 `p_ω(ŝ | ℓ, ϕ(s))`，最大化目标：  
     `J_CIB = I(ℓ; s) − I(ℓ; ϕ(s))`。
     - 解码器可利用 ϕ(s) 重建 ϕ 已捕捉的信息，因此 ℓ 被迫保留 ϕ 缺失的剩余信息（如本体感觉），成为与 ϕ 互补的嵌入。
   - **策略输入替换**：  
     `π(a | s, z) → π(a | ℓ, z), ℓ ∼ q_ψ(ℓ | s)`。  
     在数据收集、训练和下游部署中一致使用 ℓ，避免策略依赖全局坐标或背景等冗余信息。
   - **优势**：CIB 为即插即用模块，不改变原始奖励或表示学习目标，仅调整策略输入通道。

### 论文的主要结论与发现

1. **技能重标记显著提升预训练数据效率**：解决了语义漂移，使得高 UTD 比（0.5）下仍稳定训练，而基线（METRA）在高 UTD 时性能下降。GenDa 在大多数环境中用更少交互达到更高覆盖率。
2. **CIB 使技能策略对全局上下文鲁棒**：在坐标偏移和视觉背景变化下，GenDa 能保持技能执行一致性，而基线（w/o CIB）策略随全局信息变化出现不稳定行为。
3. **各项组件互补**：消融实验清楚显示：移除重标记导致覆盖学习缓慢，移除 CIB 导致下游 (RS,RG) 成功率大幅下降。
4. **高维环境成功**：在 Dog（226 维状态）和 Fish 等基线几乎失败的环境中，GenDa 能够学习有意义的技能，展现可扩展性。
5. **β 参数鲁棒**：β 从 0.1 到 4.0 变化时，性能虽有一定波动但未出现崩溃，表明方法稳定性良好。

### 不足与局限

1. **单技能每 episode 假设**：算法假设整个 episode 由单一技能 z 控制，对于需要 episode 内技能切换的场景（如先探索后利用）适用性有限。作者承认可扩展为分段标签，但未实现。
2. **CIB 依赖度量感知表示 φ**：CIB 的有效性建立在 φ 已刻画时序距离的基础上；若结合非度量感知的 URL（如 DIAYN），其互补性可能减弱。
3. **对大尺度行为的偏好**：METRA 的客观函数（最大化时序距离）天然鼓励大幅移动，难以学习精细操作（如精确抓取）。作者提出未来可用 PSD 方法引入幅度条件调节。
4. **像素背景鲁棒性有限**：尽管 CIB 提供了一定鲁棒性，但面对未见过的背景颜色/渐变时性能仍下降（表 4：从 331 降到 206 vs 从 335 降到 78）。作者承认可结合数据增强进一步改进，但目前未做。
5. **缺少真实世界验证**：所有实验均在模拟器中进行，未在真实机器人上评估，其迁移效果有待考察。
6. **算力资源未公开**：未报告 GPU 型号、数量、具体训练时长，不利于复现和对比效率。
