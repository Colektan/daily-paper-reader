## 摘要
工具增强的视觉语言模型（VLM）可以通过调用外部工具解决多模态、多步骤任务，但在实践中仍然脆弱。现有工作存在两个常见缺陷。监督微调（SFT）主要基于成功轨迹构建，对工具故障后的恢复提供的信号很少，而稀疏的轨迹级强化学习奖励仅提供关于哪一步失败以及如何修复的有限指导。我们引入了ReGRPO（反思增强型组相对策略优化），这是一种在工具使用智能体中学习反思引导纠正的框架。ReGRPO从结构化的反思数据引擎开始：我们执行近似失败动作以收集基于故障的观察，然后构建反思思维三元组（错误类型、证据、修复计划），并与纠正后的动作配对用于热启动SFT。之后，我们在局部轨迹内使用组相对优势联合优化反思令牌和纠正动作，并引入反思成本项以减少不必要的反思。在GTA和GAIA上的实验表明，在相同的骨干网络和工具套件下，ReGRPO持续优于强开源基线，并在比较的开源控制器中取得了最佳结果。代码和RoT数据可在https://github.com/showlab/ReGRPO获取。

## Abstract
Tool-augmented vision-language models (VLMs) can solve multimodal, multi-step tasks by calling external tools, yet they remain fragile in practice. Existing works have two common gaps. Supervised fine-tuning (SFT) is built mostly on successful trajectories and offers little signal for recovery after tool failures, while sparse trajectory-level RL rewards provide limited guidance on which step failed and how to repair it. We introduce ReGRPO (Reflection-augmented Group Relative Policy Optimization), a framework that learns reflection-guided correction in tool-using agents. ReGRPO starts with a structured reflective data engine: we execute near-miss actions to collect grounded failure observations, then build Reflection-of-Thought triplets (ErrorType, Evidence, FixPlan) paired with corrected actions for warm-start SFT. We then optimize reflection tokens and corrective actions jointly within local trajectories using group-relative advantages, and include a reflection-cost term to reduce unnecessary reflection. Experiments on GTA and GAIA show that, under the same backbone and tool suite, ReGRPO consistently outperforms strong open-source baselines and achieves the best results among the compared open-source controllers. Code and RoT data are available at https://github.com/showlab/ReGRPO.

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
将反思视为显式的学习变量，通过结构化反思数据引擎生成“错误-反思-纠正”监督信号，再通过修改后的 GRPO 算法联合优化反思令牌和纠正动作，引入反思成本项抑制冗余反思。

### 关键技术细节

1. **结构化反思数据引擎（Structured Reflective Data Engine）**
   - **失败诱导（Failure Induction）**：基于 MAT-AGENT 的 MM-Traj 轨迹，通过扰动工具选择或参数（如偏移边界框、选择相邻列、调用不匹配工具）生成“近似失败”动作，并在沙箱中执行获得真实失败观察（如 API 异常、空 OCR、无关裁剪）。
   - **反思三元组（Reflection-of-Thought, RoT）**：使用教师 VLM（默认 GPT-4o）为每个失败动作生成结构化反思，包含三个字段：
     - **ErrorType**：失败类型（如 ToolMismatch、ArgInvalid、GroundingDrift、InfoInsufficient）
     - **Evidence**：必须引用具体观察文本或视觉证据
     - **FixPlan**：具体的纠正策略（自然语言）
   - **纠正配对**：将原始正确动作作为纠正后的动作，形成完整监督样本 `(xi, a_fail, o_fail, z_i, a_corrected)`。

2. **反思增强型 GRPO 训练**
   - **动态局部轨迹构建**：对于每个状态，采样 K 个候选局部轨迹。若初始动作成功则轨迹提前结束；若失败则生成反思 `z_i` 和纠正动作 `a^(1)_i`，形成 `(a^(0), o^(0), z, a^(1), o^(1))`。
   - **组合奖励函数**：`R(τ) = λ_exec * 1{success} - η * C(τ) + λ_val * V(τ)`。
     - `C(τ)`：反思令牌长度的惩罚项（若未反思则为0），系数 η=0.1，迫使仅当必要才反思。
     - `V(τ)`：可选的训练时验证器分数，由三个确定性子分数（计划有效性、答案一致性、证据接地性）加权计算，无需模型调用。
   - **优势计算**：组内相对优势 `A_i(k) = R(τ_i(k)) - mean(R_i)`，与标准 GRPO 一致。
   - **目标函数**：`L_ReGRPO = -E[ (1/K)∑ A_i(k) log π_θ(τ_i(k)) ] + β * KL(π_θ || π_ref)`。
   - **关键点**：反思令牌 `z_i` 作为轨迹概率的一部分被优化，梯度直接作用于反思生成。

3. **零验证器推理（Zero-Verifier Inference）**
   - 单路径推理，无需外部验证器。
   - 使用确定性触发门 `g_i = 1{ToolError ∨ EmptyObs ∨ u_i < κ_i}`，其中 `u_i` 为动作序列的平均置信度，`κ_i` 为前几步置信度均值（自适应阈值）。
   - 触发后执行一个局部反思-纠正块，最多一个块。
