## 摘要
世界动作模型（WAM）通过基于视频的世界建模为机器人动作学习提供密集监督，是视觉-语言-动作（VLA）策略的一种有前景的替代方案。现有的WAM擅长基于物理的执行，但通常缺乏基于VLM的VLA中用于分解粗略指令的显式语言级规划接口。当家庭任务涉及复杂的多步目标时，这种分解变得至关重要——粗略的用户命令需要转换为一系列细粒度的可执行子任务。同时，该领域仍然缺乏对VLA和WAM执行能力的公平真实机器人比较，因为现有系统在数据、机器人平台和任务协议上往往不同。为了解决分解差距以及受控WAM-VLA比较的需求，我们引入了DSWAM——一个用于细粒度机器人操作的双系统世界动作基础模型。DSWAM保留系统1的WAM执行器作为默认控制路径，并仅在任务分解有用时可选地激活系统2的视觉语言子任务规划器。规划器根据短期视觉历史和全局任务提示预测可执行的子任务，而WAM执行器针对每条指令或子任务执行世界感知的动作生成。执行器通过动作预测和视频联合训练进行训练，但推理时直接预测动作块，无需显式的未来视频生成。为了使该执行路径在真实机器人上实用，我们进一步集成了TensorRT加速、异步执行和实时分块（RTC），从而策略查询不会阻塞机器人控制。为了与VLA策略进行公平的真实机器人比较，我们在DeMaVLA真实世界可变形操作设置下构建并评估DSWAM，使用匹配的机器人平台、预训练数据、后训练数据和评估标准。

## Abstract
World Action Models (WAMs) provide a promising alternative to Vision-Language-Action (VLA) policies by using video-based world modeling as dense supervision for robot action learning. Existing WAMs excel at physically grounded execution, but typically lack the explicit language-level planning interface in VLM-based VLAs for decomposing coarse instructions. Such decomposition becomes important when household tasks involve complex multi-step goals, where coarse user commands need to be converted into sequences of fine-grained executable subtasks. Meanwhile, the field still lacks a fair real-robot comparison between VLA and WAM execution capabilities, since existing systems often differ in data, robot embodiments, and task protocols. To address both the decomposition gap and the need for a controlled WAM-VLA comparison, we introduce DSWAM, a Dual-System World Action Foundation Model for fine-grained robot manipulation. DSWAM keeps a System 1 WAM executor as the default control path and optionally activates a System 2 vision-language subtask planner only when task decomposition is useful. The planner predicts executable subtasks from short-term visual history and a global task prompt, while the WAM executor performs world-aware action generation for each instruction or subtask. The executor is trained with action prediction and video co-training, but inference directly predicts action chunks without explicit future video generation. To make this execution path practical on real robots, we further integrate TensorRT acceleration, asynchronous execution, and real-time chunking (RTC) so that policy queries do not block robot control. To provide a fair real-robot comparison with VLA policies, we build and evaluate DSWAM under the DeMaVLA real-world deformable manipulation setting with matched robot platform, pretraining data, post-training data, and evaluation criteria.

- **问题**：现有世界动作模型（WAM）擅长基于物理的执行，但缺乏类似 VLM 的显式语言级规划接口，无法将粗粒度用户指令（如“整理桌子”）分解为细粒度可执行子任务（如“捡起玩具车放入盒子”“将面包放在盘子上”）。同时，该领域缺乏在相同机器人平台、数据、任务协议下对 WAM 与 VLA 策略的公平真实机器人比较。
- **目标**：提出一种既能保持 WAM 物理执行优势，又能按需提供子任务分解能力的双系统模型，并在严格匹配的设置下与 VLA 基线进行公平对比。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **双系统架构**：System 1（WAM 执行器）作为默认控制路径，负责世界感知的动作生成；System 2（可选视觉语言子任务规划器）仅在指令需要分解时启动，将粗指令拆解为连续的子任务。
- **训练-推理解耦**：执行器训练时使用视频共训练（video co-training）作为世界建模监督，但推理时直接预测动作块，不生成未来视频，降低延迟。
- **真实部署优化**：集成 TensorRT 加速、异步执行与实时分块（RTC），使策略查询不阻塞机器人控制。

### 关键技术细节
#### System 2 子任务规划器
- 基于 Rynnbrain4B 风格视觉语言模型。
- 输入：短时视觉历史 \(X_t\)（最近 5 帧，1 Hz 采样）与全局任务提示 \(p\)。
- 输出：下一个子任务指令 \(s_t\)。
- 训练数据：从机器人轨迹视频中分割出子任务区间，构建过渡感知监督标签（当前子任务、边界处下一子任务、终止信号）。
- 部署时每 2 秒更新一次子任务。

#### System 1 WAM 执行器
- 编码器：将多视图图像、语言条件、本体感受编码为潜在世界特征 \(z_t\)。
- 动作预测：使用条件流匹配（conditional flow matching）生成动作块 \(A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]\)。
- 训练损失：动作流匹配损失 + 视频共训练损失（对未来的潜在视觉 token 做流匹配），总体损失 \(L = L_{act} + \lambda_{vid} L_{vid}\)。
- 推理时：直接输出动作 chunk，无需显式视频生成。

#### 部署加速
- TensorRT：将视觉上下文引擎与动作去噪引擎分离编译，BF16 精度下从 198.2 ms 降至 73.8 ms，加速 2.69x。
- 实时分块（RTC）：异步策略推理，控制回环不阻塞。

## 6. 主要结论与发现

- **模拟基准**：DSWAM 在 RoboTwin 2.0 干净设置下平均成功率 92.38%，随机设置下 91.90%，优于所有对比 VLA 和 WAM 基线（如 DeMaVLA 干净 88.42%、随机 86.78%；Fast-WAM 干净 91.88%、随机 91.78%）。
- **真实折叠基准**：DSWAM（WAM-only）平均成功率 96.3%（DeMaVLA 92.5%），平均完成时间 1'44"（DeMaVLA 2'18"），尤其对裤子（SR 从 75%→90%）、裙子与毛巾（100% SR 且时间缩短）提升显著。
- **系统 2 规划器**：子任务指令相比原始粗指令，在“整理桌子”任务中将成功率从约 75.7% 提升至 100%，每轮错误从 3.53 降至 0.65，表明子任务分解可提高粗指令下的执行稳定性。
- **部署效率**：TensorRT+BFRTC 异步执行在保持 100% 成功率条件下大幅缩短完成时间（衬衫 1'28" vs 同步 1'47"，裤子 1'08" vs 1'50"），证明异步 RTC 对连续操作的重要性。

## 7. 优点

- **双系统按需解耦**：System 2 仅在需要时激活，避免对原子任务强加规划开销，兼顾简洁性与灵活性。
- **世界监督高效推理**：视频共训练提供密集物理监督，但推理时无需生成未来视频，结合 TensorRT 与 RTC 实现低延迟。
- **公平对照组设计**：在 DeMaVLA 设置下完全匹配平台、数据、协议，使 WAM 与 VLA 的对比结果归因于策略设计本身。
- **部署实用性**：实际机器人系统优化（异步执行、引擎分离编译）被作为方法论的一部分，而非事后工程。

## 8. 不足与局限

- **任务覆盖有限**：真实世界实验仅限于折叠（变形操纵），未验证在刚体、非刚体混合、精细装配等多样化任务上的泛化能力。
- **系统 2 验证场景简单**：子任务规划仅在单一排序任务（两个子步骤）上测试，未涉及更长推理链或需要动态重规划的场景（如物体被遮挡、中途变更指令）。
- **规划器依赖手工分割**：训练数据需要预先用 VLM 标注子任务边界，自动性不足；缺乏端到端从粗指令学习分解的方法对比。
- **未与更强规划器对比**：未与思维链规划、分层强化学习、或大规模 VLMs（如 GPT-4o）的实时规划方案比较。
- **计算资源披露不充分**：训练所需的 GPU 型号、数量、时长未说明，可能影响可重复性与能耗评估。
- **潜在偏差**：数据全部来自 ALOHA 风格双臂平台，跨平台/跨 embodiment（如单臂、人形）的泛化未验证。
