## 摘要
本文提出了一种模块化的、无需训练框架，用于在半结构化环境中进行零样本、语言引导的机器人操作。该架构通过将视觉-动作流水线分解为三个阶段：视觉感知、语义解释和任务执行，弥合了高层推理与低层运动学之间的鸿沟。为了克服标准视觉语言模型（VLM）固有的空间模糊性和语义幻觉，感知模块采用FastSAM和Set-of-Mark（SoM）提示来动态生成基于文本的锚点。然后，同一基础模型纯粹作为大型语言模型（LLM）运行，充当语义路由器，将无约束的人类指令转换为可验证、可重构的配置。最后，这些配置由任务编排器动态解析为MoveIt Task Constructor（MTC）以生成无碰撞轨迹。该框架在两种零样本实验设置中进行了评估：无约束的开放世界顺序操作和密集关系空间推理，在两种场景下实现了62%的端到端任务成功率，展示了其无需领域特定训练或手动坐标编程即可可靠执行复杂物理动作的能力。

## Abstract
This paper presents a modular training-free framework for zero-shot, language-guided robotic manipulation in semi-structured environments. The architecture bridges the gap between high-level reasoning and low-level kinematics by decomposing the vision-action pipeline into three stages: visual perception, semantic interpretation, and task execution. To overcome the spatial ambiguity and semantic hallucinations inherent in standard Vision-Language Models (VLMs), the perception module employs FastSAM and Set-of-Mark (SoM) prompting to dynamically generate grounded, alphanumeric visual anchors. The same foundation model then operates purely as a Large Language Model (LLM) to act as a semantic router, translating unconstrained human directives into verifiable, reconfigurable configurations. Finally, these configurations are dynamically parsed by a Task Orchestrator into MoveIt Task Constructor (MTC) to generate collision-free trajectories. The framework is evaluated across two zero-shot experimental setups: unconstrained open-world sequential manipulation and dense relational spatial reasoning, achieving a 62% end-to-end task success rate across both scenarios, demonstrating its capacity to reliably execute complex physical actions without domain-specific training or manual coordinate programming.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：人类能够通过自然语言和共享视觉上下文高效协作，但复制这种人与机器人之间的无缝共享自主性极其困难。现有方法主要分为两类：
  - **视觉-语言-动作模型（VLA）**：端到端映射像素到关节动作，但参数规模巨大、需要领域特定微调、计算资源要求高、难以本地部署、且缺乏可解释性和安全性。
  - **传统任务与运动规划（TAMP）**：如PDDL、GraphPlan，需要手动任务建模，泛化性差。
- **现有VLMs的局限**：高级场景理解能力强，但存在空间模糊性、语义幻觉、难以区分视觉相同物体（如多个相同螺丝），且对提示设计敏感，难以直接转换为精确可执行动作。
- **核心目标**：设计一个**模块化、无需训练**的框架，利用开源VLM/LLM，将自由形式自然语言指令直接转化为可执行的机器人操作，无需领域特定微调或手动坐标编程，同时保持可解释性和计算效率。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想
将视觉-动作流水线分解为**三个解耦的阶段**，每个阶段使用现成的基础模型，通过声明式任务架构（YAML）连接：

1. **视觉感知模块**  
   - **FastSAM**：对场景进行实例分割，生成逐像素掩码。  
   - **Set-of-Mark (SoM) 视觉提示**：在分割后的物体上动态覆盖字母数字唯一标识符（如“A”、“B”、“1”等），建立**共享空间词汇表**。  
   - **输出**：带有标识符的标注图像 + 物体变换帧广播到ROS2的tf树。

2. **语义解释模块**  
   - 使用**Molmo2（8B参数）**本地模型，接收两路输入：  
     - 用户自然语言指令（如“将红色螺丝刀放到簸箕上”）。  
     - 标注后的场景图像（带SoM标签）。  
   - 以**零样本**方式执行**语义路由**：生成一个符合预定义YAML schema的任务序列。  
     - YAML包含：`task_sequence`中的阶段（如`go_home`, `open_gripper`, `approach_pick`, `descend_pick`, `close_gripper`, `lift_vertical`, `approach_place`, `place`, `open_gripper`等）并绑定场景特定参数（如`pick_frame: screw_driver`）。  
   - **约束**：模型只能填充YAML的特定字段，若生成错误参数（如错误抓取帧），后续运动规划将失败，提供天然安全检查。

3. **任务规划与执行模块**  
   - **Task Orchestrator**：解析YAML，将模板实例化为具体MTC管道的阶段。  
   - **MoveIt Task Constructor (MTC)**：组合式运动规划框架，生成无碰撞、平滑轨迹。  
   - **执行**：在UR5e机器人上物理执行。

### 关键技术细节
- **SoM视觉提示**：类似人类协作中依靠共享视觉上下文，消除VLM对坐标密集提示的依赖，使其仅需读取显式标签即可理解场景。
- **YAML驱动任务泛化**：任务模板存储可重用策略，LLM生成的实例只是填充参数，实现“任务定义”与“任务执行”分离，支持零样本适应新指令。
- **MTC集成**：每个运动阶段（如`descend_pick`）参数化，由YAML指定规划器和约束，由Task Orchestrator自动绑定当前场景值。

### 算法流程（文字描述）
1. 接收RGB-D图像 → FastSAM分割 → SoM添加标识符 → 将标注图像和物体tf广播输出。
2. 结合用户自然语言指令+标注图像→ Molmo2推理 → 输出YAML任务序列（如`pick screw_driver place dustpan`）。
3. Task Orchestrator读取YAML → 替换模板变量 → 生成MTC阶段（如approach_pick: pick_frame = screw_driver） → 启动MTC规划 → 若成功则执行，若失败（如无效帧）则系统停止。

## 8. 不足与局限

1. **感知模块弱点**：  
   - FastSAM仍有部分-整体分割错误和几何相似物混淆。  
   - 依赖单视角2D分割，在严重遮挡或多层环境失效。  
2. **VLM的语言先验**：有时覆盖视觉证据，导致幻觉（如错误关联“按压”动作）。  
3. **缺乏闭环反馈**：执行阶段无实时状态监控，一旦动作失败（如抓取不稳定）无法重试或重新规划。  
4. **实验规模有限**：总共60次端到端试验，统计可信度不足；未测试真实动态干扰（如移动物体）。  
5. **缺乏与VLA基线直接对比**：仅定性表格对比，未在相同物理设置下比较成功率或轨迹质量。  
6. **运动原语静态**：当前只支持pick-and-place和按钮按压等简单原语，难以扩展到插入、堆叠等接触力敏感任务。  
7. **命令输入方式不自然**：通过屏幕显示文本而非语音，可能影响人机交互自然性。  
8. **应用限制**：仅在半结构化桌面环境验证，未测试高动态或非结构化场景（如抓取液体容器、变体积物体）。
