## 摘要
强化学习（RL）已广泛应用于序列决策，但由于与环境交互的高昂成本，其样本效率往往较低。最近有少量工作开始探索通过利用自然语言指令中表达的外部知识来提高RL效率。然而，现有的少数方法通常将整个指令视为单一的条件输入，未能考虑语言指导的阶段依赖性，尤其是在复杂环境中。本文提出了一种带有语言指令的分层强化学习（HRLLI）框架，该框架在决策过程中明确地将自然语言指令建模为动态可选的语义指导。HRLLI将指令分解为一组分段指导元素，其中每个指令片段可能在与环境交互的不同阶段变得相关。然后，通过“选择-行动”范式构建了一种新颖的分层RL策略结构：高层语义策略作为指导选择器，选择与当前状态最相关的指令片段来指导低层代理的决策，而低层策略则基于所选指导执行环境动作。两层策略同时学习，以最大化与环境交互的增强期望回报。这种设计使代理能够在交互过程中自适应地将语言指令接地为阶段特定的决策。在指令密集的RTFM基准上的实验表明，HRLLI持续优于强大的指令条件RL基线，证明了显式建模自适应指令选择显著提高了RL的有效性。

## Abstract
Reinforcement Learning (RL) has been widely applied to sequential decision-making, yet it often suffers from poor sample efficiency due to costly interactions with the environment. A limited line of recent work has started exploring improving RL efficiency by leveraging external knowledge expressed in natural-language instructions. However, the few existing approaches typically treat the entire instruction as a single conditioning input, failing to account for the stage-dependent nature of language guidance, especially in complex environments. In this paper, we propose \emph{Hierarchical Reinforcement Learning with Language Instructions (HRLLI)}, a hierarchical RL framework that explicitly models natural-language instructions as dynamically selectable semantic guidance during decision-making. HRLLI decomposes instructions into a set of piecewise guidance elements, where each instruction piece may become relevant at different stages of interaction with the environment. A novel hierarchical RL policy structure is then formulated in a \emph{Select-to-Act} paradigm: a high-level semantic policy acts as a guidance selector that selects the most relevant instruction piece to the current state to guide the low-level agent's decision, while a low-level policy executes environment actions conditioned on the selected guidance. The two-level policies are learned simultaneously to maximize augmented expected returns from interactions with the environment. This design enables the agent to adaptively ground language instructions into stage-specific decisions during interaction. Experiments on the instruction-intensive RTFM benchmark show that HRLLI consistently outperforms strong instruction-conditioned RL baselines, demonstrating that explicitly modeling adaptive instruction selection significantly improves the effectiveness of RL.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **问题**：强化学习（RL）在复杂任务中面临样本效率低下问题，现有语言指导 RL 方法通常将整个指令集编码为单一条件输入，忽略了不同指令片段在交互不同阶段的差异化相关性（**阶段依赖性**）。
- **意义**：真实场景中人类常提供多条松散的提示，每条只在特定阶段有用。缺乏对这种特性的建模会降低语言指导的效果，导致学习效率不佳。
- **论文目标**：提出一种分层 RL 框架，让智能体能够**动态选择**当前状态下最相关的指令片段来指导决策，从而更有效地利用语言知识。

## 2. 论文提出的方法论

- **核心思想**：采用“选择-行动”（Select-to-Act）分层范式。高层策略（选择器）根据当前状态从指令集中挑选最相关的一条，低层策略（执行器）在该指令引导下与环境交互若干步。
- **关键技术细节**：
  - **指令编码**：使用预训练 MiniLM（冻结）+ 可训练的 MLP 适配器，将每条指令映射为 $d=256$ 维嵌入。
  - **状态编码**：基于 txt2π 的卷积架构（token 嵌入 + BiGRU + 注意力池化 + 2层 3×3 CNN + 全局最大池化 + 线性投影），输出同样维度状态嵌入 $h_{S_t}$。
  - **选择器（高层策略）**：每 $T=10$ 步重新选择一次。计算状态嵌入与各指令嵌入的兼容性分数 $\ell_{k,i} = \text{score}_{\theta_H}(h_{S_{kT}}, h_{I_i})$，通过 softmax 得到选择分布，训练时采样，评估时取最大分数。所选指令的嵌入直接作为指导向量 $z_k$。
  - **执行器（低层策略）**：每步动作 $a_t \sim \pi_{\text{act}}(\cdot | h_{S_t}, z_k; \theta_L)$，使用 PPO 训练，目标为折扣累计增强回报（环境奖励 $r_t$ + 预测奖励 $\hat{r}_t$）。
  - **奖励模型**：MLP 网络 $\hat{r}_\psi(s_t, a_t, s_{t+1})$，用 MSE 学习环境奖励，提供密集信号。
  - **辅助奖励（高层）**：余弦相似度 $\cos(u_k, z_k)$，其中 $u_k$ 是 T 步平均状态嵌入，鼓励所选指令与后续轨迹对齐。
  - **联合训练**：高层用 REINFORCE 最大化（预测段回报 + 辅助奖励）；低层用 PPO；状态编码器同时接收高层和低层梯度更新。
- **算法流程**：见论文 Algorithm 1（编码指令 → 每 T 步选择指令 → 执行器交互 T 步 → 更新奖励模型 → 收集高层经验 → 周期更新选择器、执行器和状态编码器）。

## 6. 论文的主要结论与发现

- **主要结论**：显式地将语言指令建模为阶段相关的可选择指导，能显著提升复杂决策任务中 RL 的效率和性能。
- **具体发现**：
  - 在所有 8 种配置下，HRLLI 一致优于 txt2π 和 Reader，尤其在双怪物和自然语言/打乱指令困难场景下优势更大。
  - 共享状态编码器同时利用高层和低层梯度更新至关重要（w/o high-level encoder update 性能大幅下降）。
  - 基于余弦相似度的辅助奖励比基于价值差的奖励更有效。
  - 过渡条件奖励模型（$s_t, a_t, s_{t+1}$）最佳，去除奖励模型导致显著下降。
  - 高层 horizon T 在 5~20 时效果较好，过小（1）或过大（40）均降低性能。
  - 辅助奖励权重 $\lambda_{\text{sel}}$ 需适度，过大或过小均有负面影响。

## 7. 优点

- **方法论创新**：首次明确针对语言指导的阶段依赖性提出分层选择机制，而非简单编码整个指令集。
- **密集奖励设计**：通过预测奖励模型和辅助余弦相似度奖励有效缓解稀疏奖励问题，加速训练。
- **灵活的层次结构**：高层以段落尺度决策，低层以步尺度执行，自然对齐了不同时序的信用分配。
- **实验充分**：多维度消融和敏感性分析，验证了每个组件的必要性。
- **性能优异**：在具有挑战性的自然语言+打乱指令场景下，HRLLI 将 win rate 提升 15–30% 相对增益，逼近 Oracle MCTS。
