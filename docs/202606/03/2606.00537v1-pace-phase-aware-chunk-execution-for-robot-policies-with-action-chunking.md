## 摘要
近年来，基于视觉-语言-动作和扩散的机器人策略常采用动作分块机制，即每次策略查询预测一组未来动作序列，机器人先执行一段开环前缀后再重新查询。该接口虽提升了局部运动连续性，但部署时仍需选择执行视界：即在获取新观测前应执行预测分块中的多少动作。然而，我们的实验表明，任务成功率高度依赖于具体任务，且与执行视界呈非单调关系，因此单一固定视界并非可靠的部署规则。我们提出PACE（相位感知块执行），一种无需训练的测试时执行方法，直接从预测分块本身在线选择执行视界。PACE通过识别预测速度曲线中的低速过渡点，并将其作为候选重规划边界，利用操作轨迹的相位相关运动学结构。由于PACE仅依赖预测的动作分块，因此即插即用，无需重新训练或访问策略内部机制。我们在仿真和真实机器人场景中通过大规模实验验证了PACE。在50个RoboTwin2.0任务上，PACE将平均成功率从57.8%提升至64.2%。在双臂ALOHA和单臂Franka平台的真实机器人实验中，PACE将平均任务得分从60.7提升至77.7，平均成功率从50.7%提升至70.4%。消融实验和滚动分析表明，PACE能根据不同操作阶段自适应调整执行视界：在过渡阶段缩短执行长度，在连贯运动中保留较长执行。

## Abstract
Recent vision-language-action and diffusion-based robot policies often use action chunking, where each policy query predicts a sequence of future actions and the robot executes an open-loop prefix before re-querying. While this interface improves local motion continuity, deployment still requires choosing the execution horizon: how much of each predicted chunk should be executed before acquiring a new observation. However, our experiments show that success is strongly task-dependent and non-monotonic with respect to the execution horizon, making a single constant horizon an unreliable deployment rule. We propose PACE (Phase-Aware Chunk Execution), a training-free test-time execution method that selects the execution horizon online from the predicted chunk itself. PACE exploits the phase-dependent kinematic structure of manipulation trajectories by identifying low-speed transition points in the predicted speed profile and using them as candidate replanning boundaries. Because PACE uses only the predicted action chunk, it is plug-and-play and requires no retraining or access to policy internals. We validate PACE through large-scale evaluations in both simulation and real-robot settings. On 50 RoboTwin2.0 tasks, PACE raises the average success rate from 57.8% to 64.2%. In real-robot experiments on bimanual ALOHA and single-arm Franka platforms, PACE improves the average task score from 60.7 to 77.7 and the average success rate from 50.7% to 70.4%. Ablations and rollout-level analyses show that PACE adapts execution horizons across manipulation phases, shortening near transitions while preserving longer execution during coherent motion.

# 论文总结：PACE（Phase-Aware Chunk Execution for Robot Policies with Action Chunking）

## 1. 核心问题与整体含义（研究动机和背景）
- **问题背景**：视觉-语言-动作模型（VLA）和扩散策略常采用**动作分块**（action chunking），即每次查询预测一组未来动作序列。部署时需决定执行多少动作后再重新查询，称为**执行视界**（execution horizon）。
- **核心发现**：固定执行视界（如H=25）会导致成功率随任务高度变化且呈**非单调**关系。例如，在“Click bell”任务中，H=6时成功率最高，H=25附近下降40%，H=35又回升。因此**单一固定视界不可靠**，需要在线自适应选择。
- **研究意义**：执行视界是动作分块策略部署中的关键变量，但常被当作实现细节忽略。本文证明自适应选择执行视界可大幅提升成功率。

## 2. 方法论
- **核心思想**：利用预测动作块的速度剖面（speed profile）中的**低速过渡点**（low-speed valleys）作为自然的重规划边界，这些点对应操作中的阶段转换（如接触准备、抓取、释放）。
- **关键技术细节**：
  - **速度剖面提取**：对每个机械臂组的预测动作块计算一维速度序列，并平滑以抑制噪声。
  - **候选边界选择**：从每个臂的速度剖面中提取低速谷，要求谷的显著性（prominence）超过阈值δ_T，且谷间最小间距d_min。
  - **跨臂融合**：多臂情况下，将所有臂的候选边界取并集，选择最早的边界作为执行视界；若无有效边界，则回退到最大执行视界H_max。
  - **阈值校准**：δ_T仅从训练演示数据中校准一次，不依赖评估结果或成功标签。
- **公式/算法流程**（文字说明）：
  1. 策略预测动作块A_i（长度L）。
  2. 对每个执行臂b，计算速度剖面v_i^b并平滑得到~v_i^b。
  3. 从~v_i^b中提取满足最小间距的候选谷集合V_i^b。
  4. 对每个谷计算显著性Φ_i^b(r)，接受Φ_i^b(r) ≥ δ_T的谷。
  5. 跨臂合并所有接受边界，取最小值h_i；若为空，则h_i = H_max。
  6. 机器人执行前h_i个动作，丢弃后缀，再获取新观测并重新查询策略。
- **即插即用**：无需修改或重训练策略，仅依赖预测的动作块，无需注意力图、置信度等内部信息。

## 3. 实验设计
- **模拟场景**：
  - **Benchmark**: RoboTwin2.0，包含50个双臂操作任务（如放置、堆叠、抓取等）。
  - **策略**: 每个任务训练一个π0.5检查点（预测视界L=50）。
  - **对比方法**: 固定视界H=5（短）、H=25（中）、H=50（长），以及PACE。
  - **评估协议**: 每任务3个随机种子，每种子3批每批100 episode，共900 episode/任务；50任务总计45,000 episode。
- **真实机器人场景**：
  - RoboChallenge（公开基准）：双臂ALOHA机器人上的 stack_bowls 和 put_pen_into_pencil_case 任务（各30 trials/方法）。
  - 单臂Franka实验：place_object_on_plate 任务（5个物体变体，各20 trials/方法，共100 trials/方法）。
  - 对比基线：固定视界H=50（完整执行分块）。
  - 指标：任务得分（Score，0-100）和完整成功率（Succ.）。
- **消融与诊断实验**：
  - 相位剖面构造（关节空间 vs. 笛卡尔空间，是否平滑）。
  - 训练预测视界H_train的影响（5~50，固定评估视界H_eval）。
  - 超参数敏感性（谷间距d_min、阈值百分位ρ）。
  - 固定视界扫描（H=1~50）与PACE在匹配平均执行视界下的对比。

## 4. 资源与算力
- **训练资源**：
  - π0.5策略微调：RoboChallenge任务使用4块H200 GPU，FSDP和bfloat16精度。

- **真实**：共30+30+100=160 trials/方法，虽少于模拟，但在机器人领域属于中等规模，且覆盖不同平台和任务。

- **公平性**：同一检查点、相同观测，仅执行规则不同；固定基线使用全局H，PACE在线自适应；诊断中对比匹配平均执行视界的固定基线，控制查询频率。
- **结论**：实验设计客观、公平，消融充分，支持主要发现。

## 6. 主要结论与发现
- **PACE优于所有固定视界基线**：
  - 模拟50任务平均成功率：PACE 64.2% vs. 最佳固定H=25的57.8%（+6.4%）。
  - 真实机器人：平均任务得分60.7→77.7（+17分），平均成功率50.7%→70.4%（+19.7%）。
- **自适应执行视界有效**：PACE根据预测运动结构自动调整，在过渡期缩短执行（如接触前），在连贯运动中延长执行（如运输）。
- **增益来源**：不仅是通过改变平均执行频率，而是源自**更好的重规划时机**。匹配平均视界的固定基线仍低于PACE。
- **训练视界影响**：更长的训练预测视界（H_train）能提升固定执行时的成功率，尤其当评估视界较短时。

## 7. 优点
- **无需训练、即插即用**：仅依赖预测动作块的运动学特征，不需策略内部信息或额外模块。
- **原理简单且通用**：基于操作轨迹的相位结构，适用于单臂和双臂、仿真和真实机器人。
- **实验全面**：大规模模拟（45K episode）、真实机器人（多平台、多任务）、消融和诊断分析，统计性强。
- **诊断深入**：通过匹配执行视界对比，分离了自适应时机带来的好处，而非仅仅是平均查询频率。
- **开源友好**：方法可直接集成到现有动作分块策略中，无训练成本。

## 8. 不足与局限
- **不改变策略能力**：PACE只决定执行多少预测动作，若基础策略无法生成完成子任务的动作（如未完全打开铅笔盒），则无法挽救。
- **依赖运动学信号**：适用于操作阶段以低速过渡为特征的任务。若关键决策点不体现为速度谷（如纯视觉切换），或策略预测噪声大，则可能提供弱信号。
- **策略家族有限**：仅评估了π0.5一种VLA策略，未验证其他架构（如扩散策略、标记化动作模型等）下的泛化性。
- **真实实验规模**：受限于机器人操作成本，每方法仅30~100 trials，且仅在两个平台（ALOHA、Franka）测试，结论的泛化性需更多场景验证。
- **超参数需从训练演示校准**：阈值δ_T依赖少量演示数据，若演示质量差或任务间差异大，需重新校准。
