## 摘要
本文提出RepWAM，一种基于表示视觉-动作标记器的以表示为中心的世界动作模型（WAM）。现有WAM通常从预训练视频生成模型中继承以重建为导向的视频标记器。尽管这些标记器保留了视觉保真度，但仅靠像素重建对学习将未来预测与机器人控制相连接的指令跟随动态提供的指导有限。为解决此问题，我们探索了用于以表示为中心的世界动作建模的语义视觉-动作隐空间。具体而言，我们训练了一个表示视觉-动作标记器，将视觉输入映射为对齐的视觉和潜在动作标记。然后，我们预训练WAM，在语言指令下联合建模未来视觉状态以及连接它们的潜在动作，随后通过适应真实机器人轨迹实现闭环操作。在真实世界操作任务和模拟基准上的实验表明，RepWAM在多种操作设置中表现出色，而消融实验凸显了语义视觉-动作标记化相比以重建为导向的替代方案的价值。这些结果表明，表示视觉-动作标记化是世界动作模型的一个有前景的基础，也是迈向通用机器人策略的一步。代码和权重将在https://github.com/wdrink/RepWAM提供。

## Abstract
This work presents RepWAM, a representation-centric world action model (WAM) built on representation visual-action tokenizers. Existing WAMs typically inherit reconstruction-oriented video tokenizers from pretrained video generation models. Although these tokenizers preserve visual fidelity, pixel reconstruction alone provides limited guidance for learning instruction-following dynamics that connect future prediction with robot control. To address this, we explore a semantic visual-action latent space for representation-centric world action modeling. Specifically, we train a representation visual-action tokenizer that maps visual inputs into aligned visual and latent action tokens. We then pretrain our WAM to jointly model future visual states and the latent actions that connect them under language instructions, followed by adaptation to real robot trajectories for closed-loop manipulation. Experiments on real-world manipulation tasks and simulation benchmarks show that RepWAM delivers strong performance across diverse manipulation settings, while ablations highlight the value of semantic visual-action tokenization over reconstruction-oriented alternatives. These results establish representation visual-action tokenization as a promising foundation for world action models and a step toward generalist robot policies. Code and weights will be available at https://github.com/wdrink/RepWAM.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究背景**：世界动作模型（World Action Models, WAM）扩展了经典的世界建模，从被动的未来预测转向具身控制，需要同时预测视觉动态并推断机器人动作。然而，现有 WAM 通常直接继承自预训练视频生成模型的以重建为导向的视频标记器（reconstruction-oriented video tokenizers），如 WAN2.2 VAE。这类标记器优化像素重建，却缺乏对视觉语义和机器人动作的建模能力，导致：
  - 视觉隐空间被低层外观（背景纹理）占据，而物体身份、空间关系等任务相关语义不足。
  - 视觉隐空间与动作命令处于不同空间，需要额外的逆动力学模型（IDM）来弥合模态鸿沟，每步都带来信息损失。
- **核心问题**：世界动作模型真正需要什么样的表示？如何设计一个既能预测视觉动态又能支持动作推断的统一语义隐空间？
- **整体含义**：本文提出 RepWAM（Representation World Action Model），核心是构建一个表示视觉‑动作标记器（RepViTok），将视觉特征与冻结的视觉基础模型对齐，并学习潜在动作标记作为语义视觉状态之间的变换，从而在统一的语义空间中联合建模未来视觉状态和连接它们的潜在动作。实验证明该范式优于重建导向标记器，为通用机器人策略提供新基础。

## 2. 方法论

- **核心思想**：建立语义化的视觉‑动作联合隐空间，使视觉状态和动作诱导的变换在同一空间表示，从而让 WAM 能够更有效地学习指令跟随动态。
- **关键技术细节**：
  1. **视觉标记器（Visual Tokenizer）**：采用 ViT 自编码器，编码器将首帧分解为 16×16 块，后续帧分解为 4×16×16 管状块，经时间因果掩码和空间全注意力得到隐变量 \(z\)。解码器对称重建像素。监督损失包括：
     - 重建损失 \(\mathcal{L}_{\text{rec}} = \lambda_1\|o-\hat{o}\|_1 + \lambda_{\text{perc}}\mathcal{L}_{\text{perc}} + \lambda_{\text{gan}}\mathcal{L}_{\text{gan}}\)
     - 特征对齐损失 \(\mathcal{L}_{\text{align}} = \|\text{avg}(W_{\text{align}}z) - \text{avg}(G(o))\|_2^2\)，其中 \(G\) 是冻结的视觉基础模型（Perception Encoder），通过线性投影对齐，拉近隐变量与语义特征。
     - 总损失 \(\mathcal{L}_{\text{vis}} = \mathcal{L}_{\text{rec}} + \lambda_{\text{align}}\mathcal{L}_{\text{align}}\)。
  2. **潜在动作标记器（Latent Action Tokenizer, LAT）**：在冻结的视觉标记器之上，训练一个耦合的逆动力学模型（IDM）\(q_\phi\) 和前向动力学模型（FDM）\(f_\psi\)。对于连续隐帧 \(z_t, z_{t+1}\)，IDM 压缩过渡为潜在动作 \(\ell_t \in \mathbb{R}^{d_\ell}\)（\(d_\ell \ll d_v\)）；FDM 根据 \(z_t\) 和 \(\ell_t\) 预测传输矩阵 \(K_t\) 和残差 \(\delta_t\)，重建 \(\hat{z}_{t+1} = K_t z_t + \delta_t\)。训练目标：
     - 前向预测损失 \(\mathcal{L}_{\text{fwd}} = \sum_t \|\hat{z}_{t+1} - z_{t+1}\|_2^2\)
     - 反向一致性损失 \(\mathcal{L}_{\text{cons}} = \sum_t \|\hat{z}_t - z_t\|_2^2\)
  3. **因果世界动作模型（Causal WAM）**：基于扩散 Transformer，采用分块因果掩码，每个块包含视觉和潜在动作 token，共享注意力权重但使用独立的 FFN。语言指令通过冻结的 PLM 编码后通过交叉注意力注入。训练采用条件流匹配（flow matching）目标，对视觉部分和动作部分分别计算损失：
     \[
     \mathcal{L}_{\text{FM}} = \mathbb{E}\left[\|F_\theta^v(x_\alpha,\alpha,s_{<t}) - \dot{x}_\alpha^v\|_2^2 + \lambda_a \|F_\theta^a(x_\alpha,\alpha,s_{<t}) - \dot{x}_\alpha^a\|_2^2\right]
     \]
     其中 \(x_\alpha = (1-\alpha)\epsilon + \alpha u\)，\(\dot{x}_\alpha = u - \epsilon\)。分两阶段：先预训练在视觉 + 潜在动作上，再适应到真实机器人轨迹（使用连续的电机指令如末端执行器运动和夹爪控制）。

## 3. 实验设计

- **数据集**：
  - **预训练**：AgiBot（约 100G 视频‑动作隐 token，潜在动作由 LAT 生成）
  - **适应训练**：混合真实机器人语料，包括 AgiBot、RoboMIND、RoboCOIN、InternA1（总计约 300G token），提供连续电机命令。
- **Benchmark**：
  - **真实世界**：Franka 双臂机器人平台，三个任务：① Pick the fruits ② Push the drawer ③ Insert the test tube。各任务 50 条真实演示，微调 500 步，每任务 10 次物理 rollout 评估成功率。
  - **模拟环境**：RoboTwin 2.0，50 个任务（划分为 Easy 和 Hard 两类），评估成功率。
- **对比方法**：
  - \(\pi_{0.5}\)（VLA 模型）
  - Lingbot-VA（基于 WAN 视频生成预训练的 WAM）
  - Motus（统一潜在动作世界模型）
  - 自身两种规模：RepWAM-1.3B 和 RepWAM-5B
- **评估指标**：视觉生成质量（gFVD, PSNR, SSIM）、开环动作准确率（OLS, 阈值 0.03）、闭环执行成功率。

## 6. 主要结论与发现

- **视觉语义对齐提升显著**：RepViTok 相比 WAN2.2 VAE，在 AgiBot Eval Seen 上 gFVD 降低 9.5%，OLS 提高 5.14；在闭环 PickFruit 任务上成功率从 20% 提升至 30%，而 ViTok（仅重建）仅 10%。
- **潜在动作两阶段训练优于联合预测或不用**：Two Stages 在 Eval Seen 上 gFVD 降至 48.23（Joint Pred 为 94.25），OLS 提升至 19.87；闭环成功率 50%，远高于 w/o（30%）和 Joint Pred（20%）。
- **RepWAM 在真实世界表现出竞争力**：RepWAM-5B 在三个真实任务上都达到最佳或持平，尤其在长程（80%）和精密（60%）任务上显著优于 \(\pi_{0.5}\) 和 Lingbot-VA。
- **对视频 CFG 依赖降低**：RepViTok 语义空间使得视频 CFG=1.0 即达到最优，无需额外引导，有利于部署效率。
- **根本发现**：表示设计是世界动作模型的关键；语义化的视觉‑动作标记化能够提供更可解释、可迁移、有效的机器人操作能力。

## 7. 优点

- **表示创新**：首次将语义视觉基础模型引入视频标记器训练，使视觉隐空间富含物体身份和任务相关语义，而非仅外观重建。
- **潜在动作设计巧妙**：在语义视觉隐空间中定义潜在动作作为状态变换（传输矩阵+残差），动作与视觉自然对齐，且维数极低（\(d_\ell=4\)），便于后续映射。
- **两阶段训练范式**：先预训练在语义视觉‑潜在动作上，再适应机器人真实动作，解耦了视觉动态学习与运动映射，提升了可迁移性。
- **实验验证充分**：同时涵盖真实和模拟、多种规模、多种消融，指标多维，且对比了多种强基线。
- **效率优势**：无需显式未来帧生成（推理时仅需少量去噪步数），且视频 CFG 可设为 1.0，降低计算开销。
- **开源承诺**：代码和权重将公开，可复现。

## 8. 不足与局限

- **真实世界实验规模有限**：仅 3 个任务，每个仅 10 次 rollout，统计显著性有限，且未报告不同随机种子下的方差。需要更多任务和重复以证明泛化性。
- **模拟 Benchmark 上未超越最强基线**：在 RoboTwin 2.0 上，RepWAM-5B 的 Easy/Hard 得分（89.3/88.4）仍低于 Lingbot-VA（92.9/91.6），作者将其归因于缺乏 WAN 预训练，但这也表明纯语义对齐不一定胜过大规模视频生成预训练。
- **未评估零样本/小样本能力**：所有实验均使用了任务特定的微调（50 条演示），未展示预训练模型在新场景或新物体上的零样本表现。而 DreamZero 等强调零样本泛化。
- **潜在动作的语义可解释性未定量验证**：虽然展示了可视化，但缺乏定量指标（如交互检测准确率）来证明潜在动作确实编码了操作相关变化。
- **未讨论多模态融合**：仅使用视觉和语言，未融合触觉、力觉等人类操作中常用的模态。
- **资源需求高**：从零训练 5B 模型需要 128 块 H20 GPU，对一般研究组门槛较高。虽然声称从零训练，但 128 块 H20 算力仍昂贵。
- **潜在动作尺度敏感性**：\(d_\ell=4\) 的选择是否最优？文中未提供超参扫描。
- **领域限制**：仅针对桌面机器人操作（pick, push, insert），未扩展到移动操作、人机交互等更开放场景。
