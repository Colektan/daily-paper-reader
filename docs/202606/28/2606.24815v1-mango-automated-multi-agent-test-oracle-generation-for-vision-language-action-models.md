## 摘要
视觉-语言-动作（VLA）模型是一类新兴的机器人控制系统，它将感知、语言理解和动作生成集成在统一架构中。现有的VLA机器人测试方法依赖于手动构建的符号测试预言，这些预言从最终环境状态判断任务是否成功。此类预言构建成本高，需要领域专业知识，且通常与特定任务和环境紧密耦合，限制了可扩展性和复用性。此外，它们仅提供任务结果的终态评估，对中间行为和故障定位的洞察有限。为解决这些限制，我们提出MANGO，一个多智能体框架，能够从机器人任务的自然语言描述中自动生成细粒度预言。MANGO首先生成一个可复用的原子任务库，然后为每个原子任务生成基于模拟器的预言定义，最后通过将复杂指令分解为有序的原子动作序列及对应预言，生成可执行的细粒度预言。该框架使用协作的生成器、评估器和裁判智能体，通过结构化反馈迭代优化生成的工件。我们在LIBERO_10和RoboCasa Humanoid Tabletop基准上评估MANGO。结果表明，MANGO生成了可执行的细粒度预言，能够检测到与符号预言相似数量的失败，同时精确定位失败并提供更丰富的诊断信息。通过消融研究，我们进一步分析了各组件的贡献以及初始任务集的影响，同时保持了预言质量。总体而言，结果证明了面向VLA机器人测试的测试预言生成的可行性和有效性。

## Abstract
Vision-Language-Action (VLA) models are emerging robotic control systems that integrate perception, language understanding, and action generation in a unified architecture. Existing testing approaches for VLA-enabled robots rely on manually constructed symbolic test oracles that determine task success from final environment states. These oracles are costly to construct, require domain expertise, and are often tightly coupled to specific tasks and environments, limiting scalability and reuse. Furthermore, they provide only end-state assessments of task outcomes, offering limited insight into intermediate behavior and fault localization. To address these limitations, we introduce MANGO, a multi-agent framework that automatically generates fine-grained oracles from natural-language descriptions of robotic tasks. MANGO first generates a reusable library of atomic tasks, then generates simulator-grounded oracle definitions for each atomic task, and finally produces executable fine-grained oracles by decomposing complex instructions into ordered sequences of atomic actions and corresponding oracles. The framework uses collaborative Generator, Assessor, and Judge agents that iteratively refine generated artifacts through structured feedback. We evaluate MANGO on the LIBERO_10 and RoboCasa Humanoid Tabletop benchmarks. Results show that MANGO generates executable, fine-grained oracles that detect a similar number of failures as symbolic oracles while accurately localizing them and providing richer diagnostic information. Through ablation studies, we further analyzed component contributions and the effect of initial task set, while preserving oracle quality. Overall, the results show the feasibility and effectiveness of test oracle generation for VLA-enabled robots testing.

## 1. 论文的核心问题与整体含义（研究动机和背景）

* **核心问题**：现有VLA（Vision-Language-Action）机器人测试方法依赖手动构建的符号测试预言（Symbolic Test Oracle），这种预言仅通过最终环境状态判断任务成败，存在三大缺陷：
  * **构建成本高**：需要领域专家手动编写，与特定任务和环境紧密耦合，难以复用和规模化。
  * **诊断能力差**：仅提供二元的成功/失败判定，无法区分不同原因导致的失败（如“无法打开冰箱” vs “抓取瓶子失败”从终态看无区别）。
  * **缺乏中间状态评估**：无法追踪中间子任务的执行情况，阻碍了故障定位和调试。
* **研究意义**：随着VLA机器人执行长时域、多步骤的复杂任务，缺乏细粒度、自动化的测试预言已成为阻碍其可靠性和可调试性的关键瓶颈。MANGO旨在解决这一“测试预言问题”（Test Oracle Problem），实现从自然语言指令到可执行细粒度测试预言的自动化生成。

## 2. 论文提出的方法论

### 核心思想
提出**细粒度测试预言（Fine-Grained Oracles）**的概念，将复杂指令分解为有序的**原子任务（Atomic Tasks）**，并为每个子任务关联一个可重用的符号预言。通过多智能体协作框架，自动完成从自然语言到可执行预言的生成。

### 关键技术细节
* **细粒度预言的形式化定义**：
  * 指令 I 被分解为参数化原子任务序列：`D(I) = <a0(θ0), a1(θ1), ..., an-1(θn-1)>`
  * 每个原子任务有对应的符号预言：`Oa(θ) : s → {Pass, Fail}`
  * 细粒度预言由原子预言组合而成：`OF(I) = Vni=1 Oai`，并支持排序约束（`≺`表示严格顺序，`∥`表示可并行）。
* **三阶段流程**：
  1. **原子任务库生成**：从初始复杂任务集出发，由生成器（World Model）产生候选库，经评估板（多个基础LLM分别评估原子性、冗余性、重复性）和裁判（推理LLM）迭代优化。
  2. **原子任务预言库生成**：利用模拟器函数，为每个原子任务生成其对应的符号预言定义。此过程通过生成器和“Oracle Hacker”之间的辩论迭代完善。
  3. **细粒度预言生成**：针对每个新任务，结合可用对象和已有库，生成可执行的细粒度预言（包含有序步骤和对应预言逻辑表达式）。同样由评估板（逻辑、合规、接地、执行可靠性）和裁判迭代调整。
* **多智能体架构**：
  * **Generator**：使用World Model，负责生成候选方案（原子库、预言、细粒度计划）。
  * **Assessor**：使用轻量级基础LLM，并行评估特定方面（如逻辑、接地）。
  * **Judge**：使用推理LLM，综合评估反馈，做出接受/拒绝的最终裁决，并输出细化指令。

### 算法流程（文字说明）
1. **输入**：机器人任务的初始自然语言描述集合。
2. **Module I**：迭代生成并验证一个可重用的原子任务库。
3. **Module II**：迭代生成并验证一个原子任务预言库（连接原子任务和模拟器函数）。
4. **Module III**：对每个新指令，迭代生成一个可执行的细粒度预言JSON文件，包含步骤序列、执行模式和每一步的`handle_as`（对应预言的逻辑表达式）。
5. **输出**：可直接在模拟器中运行的细粒度测试预言。

## 6. 论文的主要结论与发现

* **RQ1**：MANGO在生成能力上**显著优于**所有基线。它能够生成可执行性（79%-93% vs 基线0%-11%）和结构相似度最高的细粒度预言。虽然单次生成耗时更长，但这是离线操作，可以接受。

* **RQ3**：消融实验证明**所有组件都至关重要**：
  * 移除原子任务库生成器会导致预言丢失结构，退化为终态检查。
  * 移除原子预言库生成器会导致无法将原子任务与模拟器函数正确关联，可执行性大降。
  * 移除执行可靠性检查会导致预言结构正确但无法运行。
* **RQ4**：初始复杂任务集可以**大幅缩减**（LIBERO减少69%，RoboCasa减少85.4%，平均仅需3-4个任务），而生成的原子库在语义上仍完全相同（相似度=1.0）。这表明原子任务库具有良好的通用性和复用性。

## 7. 优点

* **自动化水平高**：首次实现了从自然语言到可执行、细粒度测试预言的**端到端自动化生成**，极大减少了人工构建成本。
* **诊断能力强**：细粒度预言不仅能判断任务成败，还能**精确定位故障发生的子步骤**，并提供结构化诊断信息，是传统符号预言的重大改进。
* **高效与可复用**：原子任务和原子预言库是**一次构建，多次复用**的，能够高效支持同一基准下的大量新任务。
* **架构设计巧妙**：多智能体间的协作（生成-评估-裁判）以及迭代细化机制有效提升了生成质量的稳定性和准确性，并具备处理复杂逻辑和约束的能力。

## 8. 不足与局限

* **依赖模拟器质量**：生成的预言高度依赖底层模拟器函数的**正确性和表达能力**。在RoboCasa基准上定位准确率（70%）不如LIBERO（100%），主要是因为前者模拟器物理引擎不稳定，函数不够精准。

  * 使用**单一型号的GPU**进行所有实验，未探讨不同硬件对生成性能的影响。
  * 虽然对比了多模型，但**未深入分析**不同模型能力差异对系统性能的具体影响机制。
