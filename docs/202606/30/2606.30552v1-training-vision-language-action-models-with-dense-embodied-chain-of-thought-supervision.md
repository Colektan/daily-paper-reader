## 摘要
跨实体迁移在视觉-语言-动作（VLA）模型中仍然具有挑战性，因为不同机器人平台的低级状态和动作空间本质不同。我们观察到，操作背后的高级认知过程，包括场景感知、物体识别、任务规划和子任务分解，在很大程度上是跨实体共享的。基于这一观察，我们提出了ZR-0，一个26亿参数的端到端VLA模型，它使用密集的具身思维链（ECoT）监督来对齐视觉-语言模型（VLM）中的跨实体表示。ZR-0采用双流架构：一个预训练的VLM（系统2）在训练期间生成结构化的ECoT推理，而一个基于扩散Transformer的动作专家（系统1）通过流匹配生成连续的动作片段。这两个组件通过交叉注意力耦合，并带有一个注意力掩码，限制动作专家仅输入提示特征，从而在推理时可以完全跳过ECoT生成而没有任何性能损失。ZR-0在ProcCorpus-60M上预训练，这是一个大规模数据集，包含约6000万帧（约1000小时）来自超过40万条轨迹的数据，密集的ECoT注释覆盖了96.8%的帧。我们在三个模拟基准上评估了ZR-0，涵盖单臂（LIBERO）、双臂（RoboTwin 2.0）和类人（RoboCasa GR-1 Tabletop）实体，以及在xArm平台上的真实世界实验，展示了在所有设置下的强大性能。代码和模型检查点可在https://github.com/RUCKBReasoning/ZR-0获取。

## Abstract
Cross-embodiment transfer in vision-language-action (VLA) models remains challenging because low-level state and action spaces differ fundamentally across robot platforms. We observe that the high-level cognitive process underlying manipulation, including scene perception, object identification, task planning, and sub-task decomposition, is largely shared across embodiments. Based on this observation, we present ZR-0, a 2.6 billion parameter end-to-end VLA model that uses dense Embodied Chain-of-Thought (ECoT) supervision to align cross-embodiment representations within the vision-language model (VLM). ZR-0 adopts a dual-stream architecture: a pre-trained VLM (System 2) generates structured ECoT reasoning during training, while a Diffusion Transformer-based action expert (System 1) produces continuous action chunks via flow matching. The two components are coupled through cross-attention, with an attention mask that restricts the action expert to input prompt features only, enabling ECoT generation to be entirely skipped at inference without any performance loss. ZR-0 is pre-trained on ProcCorpus-60M, a large-scale dataset comprising approximately 60 million frames (approximately 1,000 hours) from over 400K trajectories, with dense ECoT annotations covering 96.8% of all frames. We evaluate ZR-0 on three simulation benchmarks spanning single-arm (LIBERO), bimanual (RoboTwin 2.0), and humanoid (RoboCasa GR-1 Tabletop) embodiments, as well as real-world experiments on the xArm platform, demonstrating strong performance across all settings. Code and model checkpoints are available at https://github.com/RUCKBReasoning/ZR-0.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：跨实体（cross-embodiment）迁移在视觉-语言-动作（VLA）模型中极具挑战。不同机器人平台的低级状态和动作空间（如自由度、控制接口、传感器设置）本质不同，现有方法仅通过零填充、归一化等格式级技术处理，未能解决深层的语义对齐问题，导致模型学到的表示仍然是实体特定的，难以共享和迁移。
- **关键观察**：尽管低级空间不同，但操作背后的高级认知过程（场景感知、物体识别、任务规划、子任务分解）在很大程度上是跨实体共享的。例如，无论机械臂有6个还是7个自由度，拿起杯子时遵循相似的认知轨迹。
- **整体目标**：构建一个能够利用共享的高级认知推理来对齐跨实体表示的VLA模型，实现有效的跨实体迁移，同时不牺牲推理效率。

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出ZR-0，一个26亿参数的端到端VLA模型，采用双流架构（System 1/System 2认知框架），利用密集的具身思维链（Embodied Chain-of-Thought, ECoT）监督信号来对齐VLM中的跨实体表示。
- **关键技术细节**：
  - **双流架构**：
    - **System 2（VLM）**：预训练视觉语言模型（基于Qwen3-VL-2B-Instruct），处理任务指令和图像观测，在训练时生成结构化的ECoT推理序列。ECoT包含六个组件：场景描述、进度评估、未来计划、待做动作（原子子任务）、目标物体边界框、离散动作令牌。
    - **System 1（动作专家）**：基于扩散Transformer（DiT）的模型，通过流匹配预测连续动作片段（chunk）。包含状态编码器、动作编码器、DiT块堆叠、动作解码器。
  - **跨注意力耦合**：动作专家通过跨注意力层与VLM特征交互。关键设计：使用注意力掩码，限制动作专家仅关注VLM输入提示（任务指令和图像）的特征，而**忽略ECoT令牌**。这使得推理时可以完全跳过ECoT的生成（无需自回归解码），既获得了ECoT的表示学习优势，又没有推理开销。
  - **训练数据**：使用ProcCorpus-60M，包含约6000万帧（约1000小时），来自超过40万条轨迹，ECoT标注覆盖96.8%的帧。数据来自多个开源机器人数据集（DROID、Bridge、Fractal、RH20T、Open X-Embodiment等）。同时混合通用视觉语言数据（CapsFusion、Pixmo）进行协同训练，防止VLM灾难性遗忘。
  - **训练目标**：联合优化两个损失：
    - 下一个令牌预测损失（L_ntp）用于ECoT推理。
    - 流匹配去噪矢量场预测损失（L_fm）用于连续动作生成。
    - 总损失 L = L_ntp + α L_fm，α为平衡权重（预训练时α=5，微调时为1）。
  - **推理流程**：初始化含噪声动作片段，通过N步欧拉积分逐步去噪，得到最终动作片段。整个过程不生成ECoT文本，单个前向传播即可完成，延迟约90ms（A6000 GPU）。

## 6. 主要结论与发现

- ZR-0在三个模拟基准和真实世界中均达到强性能：LIBERO 97.8%平均成功率，RoboCasa GR-1 69.3%（超越次优方法6.1个百分点），RoboTwin 2.0 Clean 88.70%/Randomized 87.98%（与LingBot-VLA相当但预训练数据量小一个数量级）。
- 真实世界任务中，ZR-0在Push Blocks（OCR推理）和Clean Table（长视距规划）上大幅优于π0.5，归功于ECoT和VL数据协同训练保留了VLM的感知和推理能力。
- 消融实验表明，移除ECoT监督导致LIBERO性能明显下降（97.8%→95.7%），验证了ECoT对跨实体表示对齐的关键作用。
- 在RoboCasa上，拾取-放置技能表现强，但关闭柜门等低频技能表现弱，说明预训练数据中技能分布对迁移效果有重要影响。

## 7. 优点

1. **创新方法**：将密集ECoT监督引入VLA训练，利用跨实体共享的高层推理实现表示对齐，思路新颖，且通过注意力掩码设计避免了推理开销。
2. **架构设计合理**：双流系统（VLM+DiT专家）灵活性高，流匹配动作生成高效，且支持可变动作维度（通过零填充和掩码损失）。
3. **大规模数据与标注**：ProcCorpus-60M提供密集ECoT标注（96.8%帧覆盖率），是现有最大规模具身思维链数据集。
4. **实验覆盖全面**：涵盖三种仿真实体和真实机器人，任务多样性好，对比方法众多，结果有说服力。
5. **开源与可复现**：代码和模型检查点已公开。
