## 摘要
我们提出了统一运动-动作模型（UMA），该方法利用三维物体运动轨迹作为共享接口，桥接了视觉运动控制和动力学建模。UMA将物体运动与机器人动作视为在掩码生成目标下共同演化的变量，其中掩码模式决定了预训练期间的监督机制和部署时的推理模式。通过事后重标定的运动上下文以及对比目标（将任务意图与场景几何解耦），UMA能够在无需人工标注任务指令的情况下，跨异构数据源进行多任务预训练。在部署时，相同的预训练参数支持运动条件视觉运动控制、基于运动的动力学建模以及从少样本演示中适应任务。在机器人演示、人类视频和模拟数据的混合数据集上预训练后，UMA在每种推理模式下都持续优于针对该模式专门设计的最新基线方法。

## Abstract
We present Unified Motion-Action (UMA) Model, an approach that uses 3D object motion trajectories as a shared interface to bridge visuomotor control and dynamics modeling. UMA treats object motion and robot actions as co-evolving variables under a masked generative objective, in which the mask pattern determines both the supervision regime during pretraining and the inference mode at deployment. Using hindsight-relabeled motion contexts and a contrastive objective that disentangles task intent from scene geometry, UMA enables multi-task pretraining across heterogeneous data sources without requiring manually annotated task instructions. At deployment, the same pretrained parameters support motion-conditioned visuomotor control, motion-based dynamics modeling, and task adaptation from few-shot demonstrations. Pretrained on a mixture of robot demonstrations, human videos, and simulated data, UMA consistently outperforms state-of-the-art baselines specialized for each inference mode.

- **问题**：现有机器人基础模型（如视觉-语言-动作模型、世界模型）在处理异构数据源（机器人演示、人类视频、仿真数据）时，通常需要手动标注任务指令，且缺乏能够同时支持视觉运动控制、动力学建模和任务适应的统一表征。
- **动机**：3D物体运动轨迹（如物体表面关键点的3D空间路径）具有与机器人动作共享坐标系的特性，可以从无动作标签的视频中提取，且在不同机器人和视角间保持可比性，但现有方法将其固定在单一角色（如仅用于运动条件策略或仅用于动力学预测），无法同时利用所有数据源进行联合预训练。
- **整体含义**：提出了UMA（Unified Motion-Action）模型，以物体运动作为共享接口，将物体运动和机器人动作视为共同演化的变量，通过掩码生成目标实现异构数据源（含/不含动作标签）的统一预训练，并在部署时支持多种推理模式（运动条件控制、动力学建模、软提示适应）。

## 2. 方法论

### 核心思想
- 将物体运动轨迹（3D关键点序列）与机器人动作序列作为条件生成模型的共同输出，通过掩码模式决定每个数据源的监督方式（仅运动 / 运动+动作），从而无需手动标注任务指令。
- 引入**任务潜在变量**c，编码抓取任务意图，并利用对比学习使其对场景几何和参考运动的具体采样方式保持不变。

### 关键技术细节
1. **运动-动作联合建模**：
   - 使用掩码扩散变压器（Masked DiT）对随机掩码的运动和动作令牌进行联合去噪，基于流匹配（Flow Matching）目标。
   - 掩码模式（G_x, G_a, P_x, P_a）决定哪些令牌作为条件（给定）、哪些需预测。
2. **任务编码器**：
   - 输入：初始观测o0（点云+视觉特征）、参考运动xc（稀疏3D关键点轨迹）；输出：M个任务令牌（构成潜在变量c）。
   - 使用PointNet++处理3D点云，参考运动的时间索引归一化到[0,1]以实现时间不变性。
3. **对比学习目标**：
   - SimCLR风格：对同一任务的不同增强视图（关键点子采样、时间子采样、SE(3)变换）拉近，对不同任务/不同运动意图的样本推远。
   - 强迫c只包含任务意图，不包含场景几何或参考运动的具体细节。
4. **预训练损失**：
   - L = λx Lx + λa La + λc Lc，其中Lx/La分别为运动/动作的流匹配损失，Lc为对比损失。
5. **推理模式**：
   - **运动条件视觉运动控制**：给定目标参考运动，生成未来动作。
   - **动力学建模**：给定候选动作序列，预测物体运动。
   - **软提示适应**：固定模型权重，仅优化任务令牌c（从少量演示中学习）。

## 3. 实验设计

### 数据集与场景
- **预训练数据**：
  - 真实机器人演示：DROID（约1000万步）。
  - 人类视频：HOI4D、Xperience（约1000万步，无动作标签）。
  - 仿真数据：基于Franka Research 3的随机操控轨迹（约1000万步，含运动-动作对）。
  - 所有预训练数据与测试环境无重叠。
- **基准测试场景**：真实世界桌面操控任务（DROID设置）：
  - **Insertion（插入）**：将餐具插入架中。
  - **Sweeping（扫除）**：用刷子将纸屑扫入簸箕。
  - **Folding（折叠）**：折叠牛仔裤。
  - 每任务20次随机试验。

### 对比方法
- **零样本运动控制**：COIL（3D运动条件策略）、UVA（联合视频-动作掩码模型）。
- **动力学预测**：PointWorld（动作条件关键点动力学模型）。
- **少样本适应**：π0.5（LoRA微调）、UVA（微调）。

### 消融实验
- 移除仿真数据、移除人类视频、移除对比学习、不同架构变体（仅动作、分离头、稠密注意力）。

## 5. 实验数量与充分性

- **零样本运动控制**：3个任务 × 20次 = 60次试验。

- **消融**：5组（w/o Sim, w/o Human, w/o CL, Action-only, Separate Heads, Dense Attention），仿真任务100次/组。

## 6. 主要结论与发现

- UMA在零样本运动控制上**超越最强基线20-25个百分点**（成功率从~55%提升至~80%）。
- 在动力学预测上MSE降低22%（0.054→0.042），移除仿真数据导致MSE激增5倍。
- 少样本适应中，UMA在动作监督下匹配/超越LoRA微调的π0.5，在运动监督（仅人类视频）下超越UVA约25个百分点。
- 消融表明：对比学习对任务适应至关重要（移除后成功率下降30-60%）；仿真数据增强运动-动作耦合；人类视频提供任务多样性。
- 失败主要源于执行精度（82%），而非任务理解或抓取。

## 7. 优点

- **统一框架**：同一个模型、同一套参数同时支持运动控制、动力学预测和任务适应，无需针对每种模式重新训练。
- **无需手动任务指令**：通过事后重标定的参考运动和对比学习自动提取任务意图。
- **异构数据利用**：能够从无动作标签的人类视频中学习运动动力学，同时利用动作标签加强控制。
- **对比学习解耦设计**：巧妙地将任务意图与场景几何分离，提升零样本和少样本泛化能力。
- **实验设计全面**：包含零样本、少样本、消融、失败分析，验证了各组件贡献。

## 8. 不足与局限

- **依赖特定实施例的动作头**：虽然运动表示通用，但动作预测头需要与机器人兼容的数据进行训练，无法直接跨实施例零样本迁移。
- **需要相机标定**：所有表示在相机坐标系下，部署时需将动作转换到机器人基座坐标系，增加了部署成本。
- **仅使用单步观测**：未利用多步历史观测或密集像素对应，而这些可自然融入掩码框架提供更强监督。
- **实验范围有限**：仅在桌面操控任务上验证，未涉及移动机器人、复杂接触、多物体场景；且仅使用一种机器人（Franka）。
- **计算资源较大**：600M参数模型需多GPU训练，可能限制复现和应用。
- **对比基线选择**：部分基线（如π0.5）在少样本适应时需完整模型微调，计算不公平，但UMA仅优化少维任务令牌，更具效率。
