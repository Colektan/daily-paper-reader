## 摘要
我们提出了统一运动-动作（UMA）模型，该方法利用3D物体运动轨迹作为共享接口，连接视觉运动控制和动力学建模。UMA将物体运动和机器人动作视为在掩码生成目标下共同演化的变量，其中掩码模式决定了预训练期间的监督机制和部署时的推理模式。利用事后重标注的运动上下文和对比目标（该目标将任务意图与场景几何分离），UMA能够在无需人工标注任务指令的情况下，跨异构数据源进行多任务预训练。在部署时，相同的预训练参数支持运动条件的视觉运动控制、基于运动的动力学建模以及从少量演示中的任务适应。在机器人演示、人类视频和模拟数据的混合集上预训练后，UMA在每种推理模式下均持续优于专门针对该模式的最先进基线方法。

## Abstract
We present Unified Motion-Action (UMA) Model, an approach that uses 3D object motion trajectories as a shared interface to bridge visuomotor control and dynamics modeling. UMA treats object motion and robot actions as co-evolving variables under a masked generative objective, in which the mask pattern determines both the supervision regime during pretraining and the inference mode at deployment. Using hindsight-relabeled motion contexts and a contrastive objective that disentangles task intent from scene geometry, UMA enables multi-task pretraining across heterogeneous data sources without requiring manually annotated task instructions. At deployment, the same pretrained parameters support motion-conditioned visuomotor control, motion-based dynamics modeling, and task adaptation from few-shot demonstrations. Pretrained on a mixture of robot demonstrations, human videos, and simulated data, UMA consistently outperforms state-of-the-art baselines specialized for each inference mode.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题**：现有机器人学习模型往往针对单一推理模式（如动作生成或动力学预测）进行专门设计，难以统一利用异构数据源（如无动作标签的人类视频、模拟数据和真实机器人演示）。同时，缺乏一种结构化的、与实体无关的中间表示来桥接视觉运动控制和动力学建模。
- **背景**：已有的运动条件策略（如 COIL）需要成对的运动-动作标签，无法吸收无动作视频；动力学预测模型（如 PointWorld）需要单独的下游控制器，难以直接融入动作标签。因此，缺少一个能同时从无动作视频中学习运动、并在有动作标签时利用动作标签的单一模型。
- **核心目标**：提出一种统一的模型，以 3D 物体运动轨迹作为共享接口，实现异构数据上的多任务预训练，并支持多种推理模式（动作生成、动力学预测、少样本适应），从而降低数据标注和模型定制的成本。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将物体运动（3D 关键点轨迹）和机器人动作视为共同演化的变量，通过一个掩码生成目标（masked generative objective）进行联合建模。掩码模式同时决定了预训练时的监督信号和部署时的推理模式。

### 关键技术细节
- **运动-动作建模**：建模条件分布 \(p_\theta(\mathbf{x}^{P_x}, \mathbf{a}^{P_a} | o_t, c, \mathbf{x}^{G_x}, \mathbf{a}^{G_a})\)，其中 \(o_t\) 是当前观测，\(\mathbf{x}\) 是物体运动序列，\(\mathbf{a}\) 是动作序列，\(c\) 是任务潜在变量（task latent），\(P\) 表示预测的 token，\(G\) 表示给定的 conditioning token。
- **任务编码器**：\(c = h_\psi(o_0, \mathbf{x}^c)\)，消费初始观测（点云 + SigLIP2 特征）和稀疏参考运动，通过 PointNet++ 和交叉注意力输出固定大小的任务 token 集。参考运动经过时间归一化（0~1）以保证对时间间隔的鲁棒性。
- **掩码扩散 Transformer（Masked DiT）**：使用独立的自适应层归一化参数分别处理目标 token（需去噪）和给定 token（保持干净），实现双角色 token 处理。注意力分解为空间（同时间步的运动-动作交互）、时间（同一关键点跨时间步）、上下文（目标 token 到观测和任务 token）三种模式。
- **异构训练**：通过随机掩码（跨模态、时间、空间）实现掩码自动编码。有动作标签的数据同时监督 \(L_x\) 和 \(L_a\)；无动作视频只监督 \(L_x\)。所有数据贡献 \(L_c\)（对比学习）。
- **对比目标**：SimCLR 风格，对同一任务的不同视图（关键点子采样、时间子采样、SE(3)变换）作为正样本，不同任务作为负样本，促使任务潜在 \(c\) 仅捕捉抽象任务意图，忽略场景几何细节。
- **总损失**：\(L = \lambda_x L_x + \lambda_a L_a + \lambda_c L_c\)。

### 推理模式
- **运动条件视觉控制**：给定参考运动 \(x^*\)，编码得到 \(c\)，预测未来动作 \(\hat{a}_{t:t+H}\)。
- **基于运动的动力学预测**：给定候选动作序列，预测引起的物体运动，用于模型预测控制或诊断。
- **少样本任务适应**：固定模型参数，优化任务潜在 \(c\) 以最小化少量演示上的重建损失，支持动作监督或运动监督。

## 3. 实验设计

### 数据集 / 场景
- **真实世界**：遵循 DROID 设置，使用 Franka FR3 机器人，ZED 相机。任务：Insertion（刚体插入）、Sweeping（工具扫地）、Folding（折叠布料）。每个任务 20 次试验，对象姿势随机化。
- **预训练数据源**：
  - 真实机器人演示：DROID（约 10M 步）
  - 人类视频：HOI4D、Xperience（合计约 10M 步）
  - 模拟机器人数据：基于 COIL 管线，Franka 机器人，多样资产（YCB、PartNet-Mobility 等），约 10M 步
- **零样本测试**：测试环境与预训练数据无重叠。

### 基准方法（Baselines）
- **视觉控制**：COIL（3D 运动条件流匹配策略，不预测运动）、UVA（联合视频-动作掩码模型，基于像素）。
- **动力学预测**：PointWorld（最先进的行动条件关键点动力学模型）。
- **少样本适应**：π0.5（LoRA 微调）、UVA（微调）。
- **消融变体**：w/o Sim（移除模拟数据）、w/o Human（移除人类视频）、w/o CL（移除对比学习）、Action-only（不预测运动）、Separate Heads（独立解码头）、Dense Attention（全注意力）。

### 评估指标
- 视觉控制：成功率（二进制，每个任务 20 次试验）。
- 动力学预测：均方误差（MSE，3D 关键点坐标，平方米）。
- 少样本适应：成功率（每个任务 20 次试验，25 个演示）。

## 4. 资源与算力

- **训练配置**：8 块 NVIDIA A100 GPU，批次大小 16，训练 16 个 epochs，线性预热 10000 步，学习率 1e-4，权重衰减 1e-6，梯度裁剪最大范数 1。
- **模型规模**：Transformer 6 层，隐藏维度 768，12 个注意力头。总参数量 600M（满规模）。
- **附录 E 缩放分析**：还评估了 150M 和 300M 参数变体，以及 50% 和 100% 数据量。

## 5. 实验数量与充分性

- **零样本视觉控制**：3 个真实任务 × 20 次 = 60 次测试。
- **动力学预测**：跨真实任务计算 MSE。
- **少样本适应**：2 种监督方式（动作监督、运动监督）× 3 个任务 × 20 次 = 120 次测试。
- **消融实验**：
  - 数据消融（w/o Sim, w/o Human）在零样本视觉控制和动力学预测上均有评估。
  - 对比学习消融（w/o CL）在少样本适应上评估。
  - 架构消融（Action-only, Separate Heads, Dense Attention）在模拟任务上各 100 次试验（3 个模拟任务）。
- **缩放实验**：6 种配置（数据/参数组合）在模拟任务上各 100 次试验。
- **充分性评价**：实验覆盖了主要推理模式、关键设计选择和数据源贡献，统计量合理，对比基线充分，公平性好。但真实任务数量有限（仅 3 个），且均为桌面操作，泛化性测试仍需拓展。

## 6. 论文的主要结论与发现

- **零样本视觉控制**：UMA 在三个真实任务上均超越最强基线（COIL 或 UVA）20~25 个百分点，证明了联合运动-动作建模优于仅以运动为条件的策略，且用运动表示动力学比像素更有效。
- **动力学预测**：UMA 的 MSE（0.042）优于 PointWorld（0.054），表明联合训练提升了动力学精度。移除模拟数据导致 MSE 飙升近 5 倍，显示模拟数据对运动-动作耦合的关键作用。
- **少样本适应**：无论动作监督还是运动监督，UMA 均优于对比基线（π0.5、UVA），仅优化任务潜在而非全部参数即可实现高效适应。
- **数据互补性**：模拟数据主要强化运动-动作耦合，人类视频主要增加任务多样性（提升 Folding 性能）。
- **对比学习必要性**：移除对比学习后少样本成功率下降 30~60 个百分点，证明解耦任务意图与场景几何至关重要。
- **架构消融**：联合预测运动+动作优于仅预测动作；共享注意力头优于分离头；结构化注意力优于全注意力。
- **错误分析**：82% 失败源于执行误差（而非任务意图），表明进一步扩大运动-动作配对数据是关键路径。

## 7. 优点

- **统一框架**：一个模型同时支持三种推理模式（动作生成、动力学预测、少样本适应），无需重新训练。
- **异构数据融合**：自然地吸收无动作人类视频、模拟数据和真实机器人数据，利用掩码目标实现跨模态监督。
- **任务表示解耦**：对比学习加事后重标注，使任务潜在变量仅包含抽象意图，对场景几何、关键点采样和 SE(3) 变换不变，从而实现跨场景转移。
- **高效适应**：少样本适应仅优化少量任务 token，无需更新模型参数，计算开销小。
- **结构化注意力设计**：Masked DiT 块通过双调制机制兼顾目标去噪与条件保持，注意力分解促进信息融合。

## 8. 不足与局限

- **具身依赖**：部署到新机器人仍需从兼容的机器人数据中训练动作头；动作空间不是完全独立的。
- **坐标系依赖**：所有表示均在每段视频的相机坐标系内，部署时需校准相机到基座的变换。
- **单帧观测**：当前仅使用单帧观测，未利用多帧历史或密集点对应关系（这些可以自然融入掩码框架，但未探索）。
- **实验覆盖有限**：仅测试了 3 个真实桌面任务，未见抓取/操作之外的场景（如移动操作、多机器人协同）。
- **模拟数据质量依赖**：模拟数据使用启发式动作基元生成轨迹，与真实世界分布可能存在差距。
- **计算成本**：预训练使用 8×A100 跑 16 epochs，规模中等，但模型可扩展性仍待更大数据验证。
- **泛化边界**：实验中目标对象与预训练集无重叠，但任务类型仍属简单操作；复杂长期任务性能未知。
