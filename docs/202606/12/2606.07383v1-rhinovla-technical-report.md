## 摘要
视觉-语言-动作（VLA）模型在机器人操作方面展现出强大潜力，但在边缘硬件上的实时部署仍然具有挑战性。在这项工作中，我们指出VLM的视觉和上下文标记是部署延迟的主要来源：对于以GEMM为主的投影算子，当模型维度固定时，计算量随输入标记数量线性增长。基于这一观察，我们提出了RhinoVLA，这是一种面向部署的VLA模型，与Huixi R1边缘SoC协同设计。RhinoVLA采用了标记高效的Qwen3-VL骨干网络和连续动作专家，减少了VLM侧的标记和计算负担，同时保留了预训练的多模态能力。为了支持跨机器人学习，RhinoVLA进一步引入了一个统一接口，该接口结合了视图注册表、72维物理状态-动作槽空间和机器人实例LoRA，使得异构机器人观测和动作模式能够在共享策略下对齐。在部署方面，RhinoVLA通过硬件感知编译、混合精度执行和并行视觉编码进行了优化。实验表明，RhinoVLA在相似参数规模下实现了与π0.5相当的下游性能，同时在Huixi R1上达到11.69Hz的端到端推理速度，满足了10Hz的实时闭环控制目标。该项目将开源在https://github.com/HuixiAI/RhinoVLA。

## Abstract
Vision-Language-Action (VLA) models have shown strong potential for robotic manipulation, but real-time deployment on edge hardware remains challenging. In this work, we identify VLM visual and context tokens as a major source of deployment latency: for GEMM-dominated projection operators, computation grows linearly with the number of input tokens when model dimensions are fixed. Motivated by this observation, we propose RhinoVLA, a deployment-oriented VLA model co-designed with the Huixi R1 edge SoC. RhinoVLA adopts a token-efficient Qwen3-VL backbone and a continuous Action Expert, reducing the VLM-side token and computation burden while preserving pretrained multimodal capability. To support cross-robot learning, RhinoVLA further introduces a unified interface that combines View Registry, 72D physical state-action slot space, and robotinstance LoRA, allowing heterogeneous robot observations and action schemas to be aligned under a shared policy. On the deployment side, RhinoVLA is optimized through hardware-aware compilation, mixed-precision execution, and parallel visual encoding. Experiments show that RhinoVLA achieves downstream performance comparable to π0.5 at a similar parameter scale, while reaching 11.69 Hz end-to-end inference on Huixi R1, meeting the 10 Hz real-time closedloop control target. The project will be open-sourced at https://github.com/HuixiAI/RhinoVLA.

### 2. 论文提出的方法论

#### 核心思想
- 识别 VLM token 数量是延迟的关键瓶颈，采用更紧凑的视觉 token 表示（Qwen3-VL 每图仅 64 个合并 token，而 PaliGemma 需 256 个），从源头减少计算。
- 设计统一的跨机器人接口，使异构机器人数据可在共享策略下对齐训练。
- 针对自研 R1 芯片进行算子级和系统级优化。

#### 关键技术细节
- **VLM 骨干**：Qwen3-VL-2B（2.13B 参数），固定预训练权重，仅加入 LoRA 适配器（rank=64）以适应机器人场景。
- **Action Expert**：0.4B 参数的连续动作生成模块，基于 Qwen 兼容的 Transformer 组件，使用 18 层 Transformer，以最后 18 层 Qwen3-VL KV 缓存为条件，预测 72D 统一动作空间的流匹配速度。
- **统一跨机器人接口**：
    - **View Registry**：为每张图像显式添加标签如 `[head|rgb]`，将相机角色和模态与数据集无关化。
    - **72D 物理槽空间**：定义 72 维向量，每维有固定物理含义（如 D0-D6 为左臂关节弧度），带二进制状态/动作掩码，仅监督有效维度。
    - **机器人实例 LoRA**：在 Action Expert 的每一层 FFN 中插入机器人特定的 LoRA 模块，训练时按实例路由，推理时可合并到主权重，保持统一部署图。
- **训练流程**：
    - 冻结 Qwen3-VL 骨干，联合训练 VLM LoRA、共享 Action Expert 和实例 LoRA。
    - 损失函数为掩码流匹配损失：$L_{FM} = \frac{\sum_{h,d} m_a(d) w(h,d) ||\hat{v}_\theta(h,d) - (z(h,d)-a(h,d))||_2^2}{\sum_{h,d} m_a(d) w(h,d) + \epsilon}$，其中 $m_a$ 为动作掩码，$w$ 为各槽权重。
    - 后训练阶段仅更新目标机器人实例 LoRA，其余参数冻结。
- **部署优化**（针对 R1 SoC）：
    - **编译优化**：算子级优化 FlashAttention 适配 SPM 内存；图级融合 MLP 与 Attention；运行时细粒度任务调度。
    - **混合精度**：INT8 权重 + FP16 激活，并实现融合式 W8A16 GEMM 内核（加载、反量化、计算流水化），减少内存带宽压力。
    - **并行视觉编码**：将多相机图像组成 batch 同时处理，代替顺序推理。

### 6. 论文的主要结论与发现

- RhinoVLA 在相似参数量下（约 2.53B）达到与 π0.5 相当的下游任务性能，并在 Huixi R1 上实现 11.69 Hz 端到端推理，满足 10 Hz 实时闭环要求。
- 实例 LoRA 能有效捕获机器人本体残差（如不同机器人动作掩码距离与 LoRA 残差相似度正相关），并带来轻微但一致的动作预测改善。
- 统一的 72D 槽空间和 View Registry 使得跨机器人预训练成为可能，且后训练只需少量目标机器人数据即可适配。
- 通过 token 压缩和硬件协同优化，VLA 模型可以在低功耗边缘芯片上实现实时控制。

### 7. 优点

- **算法-系统协同设计**：从性能分析中找到关键瓶颈（token 数量 → MLP 计算），据此选择 Qwen3-VL 并联合硬件优化。
- **统一的跨机器人接口**：View Registry + 72D 语义槽 + 实例 LoRA 三机制，有效解决了多源异构数据对齐难题，无需修改模型结构即可扩展新机器人。
- **部署优化全面**：编译、混合精度、并行编码三项优化累计将帧率从 5.84 Hz 提升至 11.69 Hz，优化效果清晰可靠。
- **实验设计全面**：覆盖模拟、真实、消融和效率分析，对比方法广泛，结果可信度高。
