## 摘要
视觉-语言-动作（VLA）模型为通用机器人策略提供了一种有前景的范式，但其适应性受到数据效率低下和泛化能力差的阻碍。我们认为这些瓶颈源于当前主流的直接指令到控制映射（Direct Instruction-to-Control Mapping），该方法迫使模型记忆整体轨迹而非可复用的运动模式，即基元（primitives）。我们提出PrimitiveVLA，一个将这一范式转变为以基元为中心的拆解与组装（Primitive-Centric Disassemble & Assemble）框架。在共享的多模态规范表示（Multimodal Canonical Representation, MCR）支持下，PrimitiveVLA统一了两个阶段：（1）微调阶段的拆解，使用自动化流程将演示拆解为可复用的基元；（2）推理阶段的组装，采用基于VLM的规划器和由LLM生成的切换模块实现鲁棒的闭环执行。通过将任务拆解为可复用的基元，PrimitiveVLA使得VLA模型能够学习不变的运动模式而非任务特定的轨迹。大量实验表明，我们的框架提高了数据效率，并在未见过的和长时域任务上实现了优越的零样本泛化。

## Abstract
Vision-Language-Action (VLA) models offer a promising paradigm for generalist robotic policies, yet their adaptation is hindered by data inefficiency and poor generalization. We argue that these bottlenecks stem from the prevailing Direct Instruction-to-Control Mapping, which forces models to memorize monolithic trajectories rather than reusable motion patterns, i.e., primitives. We propose PrimitiveVLA, a framework that shifts this paradigm toward a Primitive-Centric Disassemble & Assemble paradigm. Supported by a shared Multimodal Canonical Representation (MCR), PrimitiveVLA unifies two phases: (1) Fine-tuning-phase Disassembly, which uses an automated pipeline to disassemble demonstrations into reusable primitives; and (2) Inference-phase Assembly, which employs a VLM-based planner and an LLM-generated switch module for robust closed-loop execution. By disassembling tasks into reusable primitives, PrimitiveVLA enables VLA models to learn invariant motion patterns instead of task-specific trajectories. Extensive experiments show that our framework improves data efficiency and achieves superior zero-shot generalization across unseen and long-horizon tasks.

### 1. 论文的核心问题与整体含义（研究动机和背景）

当前 Vision-Language-Action（VLA）模型在微调时普遍采用 **“直接指令到控制映射”（Direct Instruction‑to‑Control Mapping）** 将高层指令直接映射到低层动作序列，导致模型倾向于记忆任务相关的场景特征，而非学习可复用的物理运动模式。这种范式存在两大瓶颈：

- **数据效率低**：每个新任务都需要大量高质量演示，已有知识难以跨任务迁移（如“打开柜子”与“打开微波炉”被视为独立任务）。
- **弱泛化能力**：面对未见过的物体‑技能组合或长时任务时，模型容易失败，表现为模仿干扰（imitating a similar training task）或完全“冻结”。

论文提出 **PrimitiveVLA**，将学习范式从整体轨迹记忆转变为 **“基元中心的拆解与组装”（Primitive‑Centric Disassemble & Assemble）**，旨在通过提取任务不变的基元模式来提升数据效率和零样本泛化能力。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

**核心思想**  
将多阶段任务轨迹拆解为一组有限的、任务无关的 **可复用物理基元**（如 Grasp、Pull、Push 等），微调阶段让 VLA 模型学习这些基元的底层运动规律；推理阶段则通过高层的基元序列规划和鲁棒的开关节流，将这些基元组装为完整任务。

**关键技术细节**

1. **基元定义**：论文定义了 11 个可复用的基元，分为三类：空间运输（Grasp、Place、Lift、Move）、接触与交互（Push、Pull、Insert、Press）、姿态调整（Twist、Tilt、Rotate）。每个基元有明确的运动学定义。

2. **微调阶段的基元拆解**
   - **基元序列推理（Primitive Sequence Reasoning）**：利用 VLM（Qwen3‑VL）根据任务指令、RGB 帧序列和基元库，输出基元的时间顺序序列，为后续分割提供语义先验。
   - **基于状态的边界分割（State‑Based Boundary Segmentation）**：将任务中涉及的所有基元定义输入 LLM（DeepSeek‑V3），生成 Python 代码 \(\phi_i\) 用于精确判定每个基元的终止时刻（如 Grasp 的终点是夹爪宽度恒定且末端抬升）。代码基于局部滑动窗口统计，避免噪声干扰。

   拆解后的基元数据经 **MCR** 处理，统一为规范输入，用于微调 VLA 模型。

3. **推理阶段的基元组装**
   - **基元规划器**：对于新任务指令，从拆解库 \(D\) 中检索语义最相似的 3 个（指令‑基元序列）对作为上下文（检索增强），结合初始观测和基元定义，让 VLM 生成当前任务的基元序列。
   - **基元开关（Primitive Switch）**：LLM 根据当前任务涉及的基元集生成轻量级 Python 开关函数 \(f_{\text{switch}}\)，通过滑动历史窗口监控状态统计数据，在满足条件时触发基元切换。采用双线程架构：执行线程输出 VLA 动作，监控线程并行评估开关条件，实现闭环控制。

4. **多模态规范表示（MCR）**
   - **语义统一**：将同一基元的所有样本统一为一条规范指令（如“grasp the object with green mask”），消除语言变异，引导模型关注运动模式。
   - **视觉兼容**：使用 SAM + Cutie 对目标对象进行追踪，得到对象中心掩码 \(M_i\)，观测变为 \(\tilde{o}_i = o_i \otimes M_i\)，同时保留任务上下文（通过掩码位置）并格式统一。

**损失函数**  
传统 VLA 优化整条轨迹的负对数似然：
\[
\mathcal{L}_{\text{task}} = -\sum_{t=1}^{T} \log \pi_\theta(a_t | o_t, s_t, l)
\]
PrimitiveVLA 改为优化基元级别的样本：
\[
\mathcal{L}_{\text{prim}} = -\sum_{i=1}^{N} \log \pi_\theta(a_i | \tilde{o}_i, s_i, c_i)
\]
其中 \(c_i\) 是规范基元指令，\(\tilde{o}_i\) 是掩码观测。

### 6. 论文的主要结论与发现

1. **性能提升**：PrimitiveVLA 在 Libero‑90 上使 OpenVLA 提升 9.2%（从 70.60% 到 79.80%），OpenVLA‑OFT 提升 5.0%，π0.5 保持竞争力（96.80%→96.20%），RLBench 提升 7.0%。
2. **数据效率**：仅用 50% 训练数据，OpenVLA + Ours 达到甚至超过 100% 数据 baseline 性能（80.30% vs 78.70%）；OFT 骨干下 50% 数据反而超越 100% 基线（88.87% vs 89.70%）。
3. **零样本泛化**：在 Libero‑90‑Novel 上，OpenVLA 从 7.38%→45.50%（6×提升），π0.5 从 56%→75.5%；在 Libero‑Long 上，π0.5 从 30.50%→80.25%。

### 7. 优点：方法或实验设计上的亮点

- **范式创新**：从“直接映射”转向“拆装基元”，理论上更接近组合泛化本质，显著提升数据效率和零样本能力。
- **全自动化拆解**：无需人工标注基元标签，利用 VLM+LLM 生成先验序列和物理规则代码，可扩展性强。
- **统一 MCR**：通过语义统一和视觉掩码，既消除上下文干扰又保留任务必要信息，是连接拆解与组装的关键。
- **鲁棒闭环执行**：基于 LLM 生成的开关函数和双线程架构，保证基元间平滑切换，容忍传感器噪声。
- **实验全面性**：覆盖多种骨干、多基准、仿真和真实，消融实验清晰解释各模块贡献，失败模式分析深入。

### 8. 不足与局限

1. **预定义基元集**：11 个基元基于标准夹爪操作，对灵巧手或特殊操作（如拧螺丝、剪切）覆盖不足，需要扩展。
2. **拆解依赖 VLM/LLM**：虽自动化，但 VLM 可能产生噪声（<5% 的合并错误、掩码漂移），虽经规则和 VLA 鲁棒性缓解，但未完全消除。
3. **具身泛化有限**：真实世界仅在 UR5e 单一平台验证，未涉及不同机械臂形态或更复杂场景（如移动操作、多人协作）。
4. **长时任务实战限制**：Libero‑Long 使用重置协议（每次子任务后复位到初始位姿），降低了 OOD 漂移，但在真实连续执行中复位策略不现实。且开关逻辑基于启发式，可能无法覆盖所有失败情况。
5. **未探索端到端可微规划**：当前高层规划（VLM）与低层控制（VLA）耦合较松散，未来可结合可微分规划以联合优化。
6. **可解释性**：虽然基元序列有语义可解释性，但 VLA 内部决策仍较黑盒，开关代码的阈值依赖人工辅助。
