## 摘要
视觉-语言-动作（VLA）模型越来越多地依赖历史、深度或4D特征来条件化机器人策略，以解决长程操作中的歧义。然而，更多的时空证据并不一定更好：当注入的证据不是运动一致时，可能会引入几何漂移、碎片化的时间线索和不稳定的动作生成。这引发了一个简单的问题：VLA应该记住过去的帧，还是记住连接它们的运动？我们提出MotionVLA，一种运动历史接口，将短的仅过去视频窗口转换为紧凑的时间连续轨迹场令牌。MotionVLA不是将历史视为一组稀疏的独立提升帧，而是将最近的观察表示为物理一致的运动证据。当前视觉令牌查询此历史以获取任务相关的运动信息，然后这些信息在轨迹地面真值监督下重新耦合到VLA流中。在模拟基准和初步真实机器人部署上的实验表明，MotionVLA改善了长程操作，同时产生更平滑和更直接的执行。这些结果表明，有效的VLA记忆不仅仅是提供更多的4D上下文，而是暴露可用于控制的运动一致证据。

## Abstract
Vision-language-action (VLA) models increasingly condition robot policies on history, depth, or 4D features to resolve ambiguity in long-horizon manipulation. However, more spatiotemporal evidence is not necessarily better: when the injected evidence is not motion-consistent, it can introduce geometric drift, fragmented temporal cues, and unstable action generation. This raises a simple question: should a VLA remember past frames, or remember the motion that connects them? We introduce MotionVLA, a motion-history interface that converts a short past-only video window into compact, time-continuous trajectory-field tokens. Instead of treating history as a sparse set of ndependently lifted frames, MotionVLA represents recent observations as physically coherent motion evidence. Current visual tokens query this history to retrieve task-relevant motion information, which is then recoupled into the VLA stream under trajectory-grounded supervision. Experiments across simulation benchmarks and preliminary real-robot rollouts show that MotionVLA improves long-horizon manipulation while producing smoother and more direct executions. These results suggest that effective VLA memory is not just about providing more 4D context, but about exposing motion-consistent evidence that is usable for control.

## 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有视觉-语言-动作（VLA）模型在长程操作中依赖历史帧或4D特征来消除歧义，但当注入的历史证据缺乏**运动一致性**（motion-consistent）时，反而会引入几何漂移、时间碎片化和不稳定的动作生成，导致执行路径绕远或动作抖动。
- **关键洞察**：有效的VLA记忆不应只是简单存储更多观测帧，而应**表示连接观测之间的运动**。作者提出“VLA应该记住运动，而不是帧”这一核心观点。
- **研究背景**：VLA模型（如π0）将动作生成视为条件序列预测；近期工作（如4D-VLA、SwiftVLA）引入额外的时空线索，但框架级独立提升或稀疏采样会导致几何不一致。本文旨在提供一种**紧凑、时间连续、可查询的运动历史接口**，使注入的证据真实服务于控制。

## 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：将短时间窗口的过去观测转化为**紧凑的时间连续轨迹场令牌**（trajectory-field tokens），作为可查询的运动历史；当前视觉令牌通过交叉注意力检索任务相关的运动证据，并在轨迹重建监督下重新耦合到VLA流中。
- **关键技术细节**：
  - **运动历史构建**：使用冻结的轨迹提取器（TraceAnything）对过去头摄像头的T帧（严格排除当前帧）提取密集轨迹场，但仅保留最终解码器令牌，并投影到VLA隐藏维度作为查询记忆。
  - **解耦-再耦合（Decouple-then-Recouple）设计**：
    1. **解耦**：当前帧视觉令牌作为查询，通过多头注意力从运动历史中检索运动条件令牌。
    2. **监督**：附加轻量头预测与历史窗口对齐的动作序列，最小化重建损失$\mathcal{L}_{\text{traj}}$，使运动令牌编码控制相关动态而非外观差异。
    3. **再耦合**：将运动条件令牌与多视图视觉令牌拼接，经MLP融合后送入VLA主干与文本、状态令牌共同自注意力。
  - **动作损失**：采用π0的流匹配行为克隆，对噪声加噪的动作回归条件速度场。
  - **训练策略**：两阶段训练——第一阶段在100小时异构操作数据上对齐运动历史接口（使用动作损失+轨迹损失），第二阶段在目标任务上微调（去掉轨迹预测头）。
- **公式与算法**（文字说明）：
  - 构建历史窗口$W_t$，用轨迹提取器得到令牌$M_{kv}$。
  - 当前视觉令牌$V_{\text{head}}$查询$M_{kv}$得到$M$（多对多注意力）。
  - $M$与多视图令牌$V_{mv}$拼接通过MLP融合得$F$。
  - $F$与文本、状态令牌共同输入VLA，操作动作损失和轨迹重建损失。

## 论文的主要结论与发现

1. **运动一致性是关键**：VLA模型性能提升不能仅靠增加历史或4D令牌，而需保证注入的证据是运动一致的。MotionVLA通过时间连续轨迹场令牌实现了这一点。
2. **成功率提升**：
   - RoboTwin2.0平均成功率53%，超过π0的41%，尤其是长程任务`blocks_touching_rgb`从19%提高到41%。
   - LIBERO-LONG成功率91.2%，优于π0的85.2%，整体平均95.4%为最优。
3. **路径效率改善**：在LIBERO-Goal/Long上PE为1.05/1.10，明显优于π0的1.23/1.19和4D-VLA的1.36/1.32，说明执行更直接。
4. **真实机器人初步验证**：提高成功率（28.7%→41.9%）并减少平均执行步数（338→282）。
5. **消融揭示设计必要性**：无第一阶段对齐时性能差；无轨迹监督或不再耦合时性能下降，证明每个组件都有贡献。

## 优点

- **方法新颖**：首次将时间连续轨迹场作为VLA的历史记忆接口，替代离散帧独立提升，从根本上解决几何不一致。
- **设计精巧**：解耦-再耦合机制使当前观察主动检索相关运动证据，避免了全部历史堆叠带来的噪声；辅助轨迹损失监督令牌动态内容。
- **评价指标全面**：不仅用成功率，还用路径效率（PE）衡量运动平滑度和直接性，避免了仅关注成功忽略质量的问题。
- **实验严谨**：进行了组件消融、参数消融、控制鲁棒性分析，并展示训练收敛效率，有力支撑结论。
- **开源与可复现**：基于π0框架，细节清晰，便于社区复现。

## 不足与局限

1. **适用范围**：运动历史在观测充分时可能中性甚至轻微有害（如简单任务），本文未测试这类场景的鲁棒性。
2. **轨迹提取器限制**：使用冻结的TraceAnything，在遮挡、快速运动、纹理缺失场景可能输出噪声，虽通过控制实验验证了容忍度，但未提出改进方案。
3. **真实实验规模小**：仅在一个平台（Agilex Piper）上的3个任务，缺乏跨机器人、多场景、多随机种子的系统评估，结论一般性待验证。
4. **效率问题**：每步需要运行轨迹提取器一次，虽通过稀疏采样控制计算量，但实时性未详细对比（附录给出FPS为7.1，其他配置更低）。
5. **超参数敏感性**：历史窗口长度和步幅需手工调节（最优Thist=5, ∆=2），未探索自适应或学习式选择方案。
6. **缺乏端到端统一训练**：第一阶段对齐与第二阶段微调分离，可能不是最优；未探讨联合训练。
