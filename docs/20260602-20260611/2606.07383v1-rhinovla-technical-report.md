## 摘要
视觉-语言-动作模型在机器人操控方面展现出强大潜力，但在边缘硬件上的实时部署仍具挑战。本工作中，我们将VLM视觉和上下文令牌识别为部署延迟的主要来源：对于以GEMM为主的投影算子，当模型维度固定时，计算量随输入令牌数量线性增长。基于此观察，我们提出了RhinoVLA，一种与慧溪R1边缘SoC协同设计的部署导向型VLA模型。RhinoVLA采用令牌高效的Qwen3-VL骨干网络和连续动作专家，减少了VLM侧的令牌和计算负担，同时保留了预训练的多模态能力。为了支持跨机器人学习，RhinoVLA进一步引入了统一接口，结合了视角注册表、72维物理状态-动作槽空间和机器人实例LoRA，使得异构机器人的观测和动作模式能够在共享策略下对齐。在部署方面，通过硬件感知编译、混合精度执行和并行视觉编码对RhinoVLA进行了优化。实验表明，RhinoVLA在相似参数量下实现了与π0.5相当的下游性能，同时在慧溪R1上达到11.69 Hz的端到端推理频率，满足了10 Hz的实时闭环控制目标。项目将开源在https://github.com/HuixiAI/RhinoVLA。

## Abstract
Vision-Language-Action (VLA) models have shown strong potential for robotic manipulation, but real-time deployment on edge hardware remains challenging. In this work, we identify VLM visual and context tokens as a major source of deployment latency: for GEMM-dominated projection operators, computation grows linearly with the number of input tokens when model dimensions are fixed. Motivated by this observation, we propose RhinoVLA, a deployment-oriented VLA model co-designed with the Huixi R1 edge SoC. RhinoVLA adopts a token-efficient Qwen3-VL backbone and a continuous Action Expert, reducing the VLM-side token and computation burden while preserving pretrained multimodal capability. To support cross-robot learning, RhinoVLA further introduces a unified interface that combines View Registry, 72D physical state-action slot space, and robotinstance LoRA, allowing heterogeneous robot observations and action schemas to be aligned under a shared policy. On the deployment side, RhinoVLA is optimized through hardware-aware compilation, mixed-precision execution, and parallel visual encoding. Experiments show that RhinoVLA achieves downstream performance comparable to π0.5 at a similar parameter scale, while reaching 11.69 Hz end-to-end inference on Huixi R1, meeting the 10 Hz real-time closedloop control target. The project will be open-sourced at https://github.com/HuixiAI/RhinoVLA.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：视觉-语言-动作模型（VLA）在机器人操作中表现优异，但在边缘硬件（如Jetson Orin）上实时部署困难，主要瓶颈是VLM骨干网络和动作专家的延迟，而VLM延迟又集中在前馈网络（MLP）的GEMM投影算子，其计算量随输入令牌数量线性增长。
- **背景**：现有VLA系统（如π0.5）在Orin上端到端推理仅约1.17 Hz，远低于10 Hz的实时闭环控制需求；同时异构机器人数据集缺乏统一输入输出接口，阻碍跨机器人学习。
- **目标**：设计一种注重部署效率的VLA模型，达到实时边缘控制（≥10 Hz）并支持跨机器人泛化。

## 2. 方法论：核心思想、关键技术细节
- **核心思想**：算法-系统协同优化。从源头减少VLM令牌数量（采用令牌高效的Qwen3-VL骨干），设计统一跨机器人接口，并通过硬件感知编译与混合精度加速在自研芯片R1上部署。
- **关键技术细节**：
  - **架构**：两模块分解，VLM骨干（Qwen3-VL-2B，2.13B参数）+ 连续动作专家（0.40B参数，18层，1024隐藏维度，条件化于VLM最后18层KV缓存）。
  - **令牌减少**：Qwen3-VL在224×224下每图像仅64个合并视觉令牌（PaliGemma为256个），降低约4倍。
  - **统一跨机器人接口**：
    - **视角注册表**：每个图像标记角色（head/left_wrist等）和模态（rgb/depth），消除相机布局异构。
    - **72维物理槽空间**：固定语义（如D0-D6为arm0关节，D58-D60为基础速度），配二进制掩码指示有效维度，解决动作模式异构。
    - **机器人实例LoRA**：在动作专家每层前馈网络中插入LoRA适配器，以捕获实体特异性残差，训练时仅激活对应实例LoRA，推理时可合并权重。
  - **训练策略**：
    - 预训练：冻结Qwen3-VL骨干，联合训练VLM LoRA、共享动作专家和机器人实例LoRA。使用掩码流匹配损失，仅监督有效动作槽。损失函数：`L_FM = Σ_{h,d} m_a(d) w(h,d) ||v̂_θ - (z-a)||² / Σ_{h,d} m_a(d) w(h,d) + ε`，其中`x_t = (1-t)a + tz`，目标为`z-a`。
    - 后训练/迁移：冻结VLM和大部分动作专家，仅更新目标机器人的实例LoRA，用小量真实机器人数据适应。
  - **部署优化**：
    - 硬件感知编译：优化注意力内核（R1 SPM上的FlashAttention风格平铺）、图级算子融合、细粒度任务调度。
    - 混合精度：INT8权重+FP16激活，定制W8A16 GEMM内核（权重加载、反量化、矩阵乘融合），达到1.69×加速（up_proj算子）。
    - 并行视觉编码：三视角图像批处理，延迟从34.52 ms降至24.31 ms。
