## 摘要
使机器人能够根据自然语言命令理解和执行任务，同时保持数据效率仍然具有挑战性。视觉-语言-动作模型和视觉-语言模型等基础模型提供了直观的交互通道，但需要大量数据；任务参数化模仿学习实现了数据效率，但缺乏自然语言基础。这项工作通过一个模块化架构弥合了这一差距，该架构将任务参数化的核化运动基元与预训练的视觉-语言模型相结合。在学习过程中，从2到5次动觉演示中获取技能，视觉-语言模型生成描述每个技能参数和前提条件的技能模式。在执行过程中，视觉-语言模型解释命令以选择技能、推理参数绑定，并通过协方差加权组合创建新颖的行为。当没有技能或组合适用时，系统识别能力缺口并请求有针对性的演示，所有这些都无需微调。在七自由度机械臂上的验证显示，在需要技能选择、组合和主动学习的场景中，成功率达到73.3%至100%。

## Abstract
Enabling robots to understand and execute tasks from natural language commands while maintaining data efficiency remains challenging. Foundation models such as vision-language-action (VLA) and vision-language models (VLMs) provide intuitive interaction channels but require extensive data; task-parameterized imitation learning achieves data efficiency but lacks natural language grounding. This work bridges this gap through a modular architecture combining task-parameterized kernelized movement primitives (TP-KMPs) with pretrained VLMs. During learning, skills are acquired from 2 to 5 kinesthetic demonstrations, and the VLM generates skill schemas describing each skill's parameters and preconditions. During execution, the VLM interprets commands to select skills, reason about parameter bindings, and create novel behaviors through covariance-weighted composition. When no skill or composition suffices, the system identifies capability gaps and requests targeted demonstrations, all without fine-tuning. Validation on a 7-DoF manipulator shows success rates of 73.3%-100% in scenarios requiring skill selection, composition, and active learning.

#### 2. 论文提出的方法论

**核心思想**:
通过一个模块化架构，将**预训练的视觉-语言模型**（负责语义理解、推理和交互）与**任务参数化核化运动基元**（负责数据高效的技能学习和执行）无缝结合。VLM 自动为 TP-KMP 学习的技能生成“模式”（即元数据描述），使 VLM 能够无需微调直接理解和使用这些技能。

**关键技术细节与流程**:

1.  **学习阶段**:
    *   **技能获取**: 用户通过 2-5 次动觉演示教机器人一个技能（如“抓取苹果并放到盘子里”）。
    *   **TP-KMP 训练**: 从演示数据中学习轨迹，并将其编码到任务相关的物体坐标系中（如抓取物体的坐标系）。这使得技能在新物体位置下也能泛化。
    *   **自动模式生成**: 视觉-语言模型接收工作区图像和检测到的物体信息，自动生成一个描述该技能的 JSON 模式。这个模式包含了：
        *   **技能名称**: 如“PickAndPlace”。
        *   **任务参数**: 需要的物体类型和它们的交互顺序（如“object_to_pick”和“object_to_place”）。
        *   **前提条件**: 如“抓手必须为空”。
        *   **示例和语义**: 为视觉语言模型提供上下文。
    *   **技能库**: 训练好的 TP-KMP 及其自动生成的模式被一起存入技能库。

2.  **执行阶段**:
    *   **命令解析**: 用户发出自然语言命令（如“将轴承环插入测量站”），视觉语言模型结合当前工作区图像和技能库中的模式进行决策。
    *   **决策树流程**:
        *   **技能匹配**: 视觉语言模型尝试将命令直接匹配到技能库中的某个技能。
        *   **组合可行性**: 如果单个技能不匹配，视觉语言模型会检查是否有两个或多个兼容的技能可以通过**协方差加权融合**组合成一个新技能。组合的可行性由数学上的**兼容性约束**（公式1）决定。
        *   **主动能力获取**: 如果以上两者都不可行，系统会主动检测到“能力缺口”，并向用户请求针对性的新演示来学习缺失的技能，然后将其加入技能库，无需微调。

3.  **技能组合**:
    *   **协方差加权融合**: 这是实现轨迹级组合的核心。每个 TP-KMP 的协方差矩阵表征了其在轨迹不同阶段的置信度。当组合两个技能时，它们的协方差自然地决定了哪个技能在哪个时间点占据主导地位（低协方差区域主导），从而实现平滑过渡。
    *   **兼容性约束** : 决定两个技能能否组合的数学条件。它要求两个技能的协方差必须在时间轴上形成互补的“主导区域”，即不能出现两者都高置信度或都低置信度的情况，否则会被拒绝组合，避免产生无意义的平均化轨迹。

4.  **主动学习**:
    *   当现有技能和所有可能的组合都无法满足用户请求时，视觉语言模型会生成一个自然语言请求，向用户解释为何无法完成任务，并请求演示新技能。这实现了技能库的持续、数据高效扩展。

*   **组合的局限性**: 技能组合目前**仅限于满足兼容性约束的成对组合**。虽然理论上可以链式组合，但随着深度增加，满足约束会越来越难。
